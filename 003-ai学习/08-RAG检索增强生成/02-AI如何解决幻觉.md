# AI 如何解决幻觉

> **一句话先说清楚**：幻觉无法被"根治"，因为它是大模型生成原理的副产品；我们能做的是**从五个层面层层设防**——让模型知道得更准（检索）、说得更稳（提示）、答得更老实（约束）、错了能被抓出来（验证）、不确定时敢说"不知道"（兜底）。

---

## 第一部分：先搞懂幻觉到底是什么（总）

### 什么是幻觉

**幻觉（Hallucination）**：模型生成了**听起来很合理、但实际上是错误或虚构**的内容。

它最危险的地方不是"错"，而是**错得理直气壮、有理有据**。

举几个真实例子：

```
问：《三体》的作者是谁？
答：《三体》的作者是刘慈欣。            ✅ 正确

问：刘慈欣2019年获得了什么诺贝尔奖？
答：刘慈欣于2019年获得诺贝尔文学奖。     ❌ 幻觉（他从未获诺奖）

问：帮我引用一下这篇论文的第3.2节。
答：根据论文第3.2节所述……（编造一段不存在的内容）  ❌ 幻觉
```

注意第二个例子的危险性：模型没有拒绝，而是**顺着你的预设编了一个答案**。

---

### 为什么会产生幻觉（根因）

要解决幻觉，必须先理解它**不是 bug，而是 LLM 工作原理的必然结果**。

```
LLM 的本质：根据前面的词，预测下一个最可能的词。
            它优化的目标是"流畅、合理"，不是"真实"。
```

具体有四个根因：

| 根因 | 解释 | 例子 |
|------|------|------|
| **概率生成机制** | 模型选的是"最可能的词"，不是"最真实的词" | 不知道答案时，它会填一个"看起来对"的 |
| **训练数据有限/过时** | 模型只知道训练截止前的知识，且数据本身可能有错 | 问 2025 年的事，它用 2023 年的知识硬答 |
| **没有"我不知道"的本能** | 训练数据里很少有"我不知道"，模型倾向于一定给个答案 | 编造而非拒答 |
| **压缩损失** | 万亿参数压缩了海量知识，细节会丢失或串味 | 把两个人的事迹张冠李戴 |

**关键认知**：
> 你不能指望"换个更聪明的模型"就消除幻觉。GPT-5、Claude Opus 也会幻觉，只是概率更低。
> 解决幻觉是**工程问题**，不是**模型选型问题**。

---

### 解决幻觉的总体思路：五层防御

把幻觉当作"水"，我们要做的是层层筑坝：

```
┌─────────────────────────────────────────────┐
│  第1层：知识注入（RAG）  →  给模型真实的依据    │
├─────────────────────────────────────────────┤
│  第2层：提示约束        →  框住模型的回答方式    │
├─────────────────────────────────────────────┤
│  第3层：生成控制        →  降低乱编的概率        │
├─────────────────────────────────────────────┤
│  第4层：事后验证        →  抓出已产生的幻觉      │
├─────────────────────────────────────────────┤
│  第5层：兜底机制        →  不确定时安全降级      │
└─────────────────────────────────────────────┘
```

下面逐层拆开讲。

---

## 第二部分：第1层 —— 用 RAG 注入真实知识（分）

### 为什么 RAG 是解决幻觉的"主力军"

幻觉的最大来源是**模型不知道、却硬答**。那最直接的办法就是：**别让它靠记忆答，让它看着资料答。**

这就是 RAG（检索增强生成）的核心价值：

```
普通 LLM：     问题 → 模型凭记忆答          （容易幻觉）

RAG：  问题 → 检索真实资料 → 把资料+问题给模型 → 模型照着资料答  （幻觉大降）
```

类比：
- 普通 LLM = 闭卷考试，靠背
- RAG = 开卷考试，看着书答

开卷考试当然更不容易答错。

---

### RAG 降低幻觉的关键细节

光接个 RAG 不等于幻觉就没了。真正起作用的是这几个细节：

**1. 让模型"只能"用检索到的内容回答**

```python
prompt = f"""请严格根据以下【参考资料】回答问题。

【参考资料】
{retrieved_context}

【问题】
{question}

要求：
1. 只能使用参考资料中的信息，不要使用你自己的知识。
2. 如果参考资料中没有相关信息，必须回答"根据现有资料无法回答"。
3. 回答时标注信息来源（第几段）。
"""
```

这段提示词里，三条要求每一条都在堵幻觉的口子。

**2. 检索质量决定上限**

如果检索回来的资料本身就是错的或不相关的，模型照着错的答，照样幻觉。所以：

- **召回要全**：相关的资料别漏（用 hybrid search：向量 + 关键词）
- **排序要准**：最相关的排前面（用 rerank 重排序）
- **噪音要少**：不相关的别混进去（设相似度阈值过滤）

> 详见本模块前面的 RAG 检索章节。这里只强调：**检索质量 = 幻觉控制的天花板**。

**3. 强制引用来源（Citation）**

要求模型在每句话后标注它依据的是哪段资料：

```
回答：刘慈欣是中国科幻作家[来源1]，代表作《三体》获2015年雨果奖[来源2]。

[来源1] 维基百科·刘慈欣条目
[来源2] 第74届世界科幻大会公告
```

这么做有两个好处：
- 用户能自己核对，不被"理直气壮的错误"骗到
- 模型被要求"指出依据"时，会更克制，不敢乱编（编了就得伪造来源，难度更高）

## 第三部分：第2层 —— 用提示工程框住模型（分）

即使没有 RAG，好的提示词也能显著降低幻觉。核心是**给模型"允许示弱"的空间**。

### 技巧1：明确授权"不知道就说不知道"

模型默认倾向于硬答，因为训练时"给答案"被奖励。你需要**显式地告诉它：不知道是 OK 的**。

```python
prompt = """回答问题时，如果你不确定或不知道答案，
请直接说"我不确定"或"我没有相关信息"，
绝对不要编造。诚实地承认不知道，比给一个错误答案更有价值。

问题：{question}
"""
```

这一句话能挡掉相当一部分幻觉。

---

### 技巧2：让模型先"找证据"再下结论（CoT）

强迫模型把推理过程写出来，它就没法"跳步乱编"：

```python
prompt = """请按以下步骤回答：
1. 列出你确定知道的相关事实
2. 指出哪些信息你不确定
3. 基于确定的事实，给出结论

问题：{question}
"""
```

把思考过程暴露出来，幻觉更容易在"第2步"被模型自己拦下。

---

### 技巧3：给角色和边界

```python
prompt = """你是一个严谨的医学信息助手。
你只回答有明确医学共识的问题。
对于有争议、需要个体诊断的问题，你必须建议用户咨询医生，而不是自行判断。

问题：{question}
"""
```

明确的边界 = 明确的"什么时候该闭嘴"。

---

### 技巧4：少样本示范（Few-shot）"如何拒答"

直接给模型看几个"遇到不知道的就拒答"的例子：

```python
prompt = """示例：
Q：地球到月球的距离是多少？
A：平均约38.4万公里。

Q：我邻居老王昨天吃了什么？
A：我无法获取这类私人信息，无法回答。

Q：{question}
A："""
```

模型会模仿这种"该答就答、该拒就拒"的风格。

---

## 第四部分：第3层 —— 控制生成过程（分）

这一层是从"采样参数"和"模型能力"上动手。

### 调低 temperature

`temperature` 控制生成的随机性：

```python
response = client.messages.create(
    model="claude-sonnet-4-6",
    temperature=0,      # 0 = 最确定，几乎不"发挥"
    messages=[...]
)
```

- `temperature` 高（如 1.0）：更有创意，但更容易乱编 → 适合写作
- `temperature` 低（如 0）：更保守、更稳定 → 适合问答、事实类任务

**事实类、知识库类任务，一律用 `temperature=0` 或接近 0。**

> 注意：低 temperature 只能减少"随机性带来的不稳定"，不能消除"模型本身就不知道"的幻觉。它是辅助手段，不是主力。

---

### 选对模型

不同模型的幻觉率确实有差异。一个实用策略是**分层**：

```python
# 事实严谨度要求高的环节，用能力更强的模型
strict_model = "claude-opus-4-5"     # 关键事实判断

# 一般对话、改写，用更快更省的模型
fast_model = "claude-haiku-4-5"
```

能力更强的模型，在"知识边界"上的自我认知也更好，更愿意说"我不确定"。

## 第五部分：第4层 —— 事后验证，抓出幻觉（分）

前面三层是"预防"，但总有漏网之鱼。第4层是"事后检查"——生成完了，再验一遍。

### 方法1：用另一个模型当"事实核查员"（LLM-as-a-Judge）

让第二个模型专门检查第一个模型的回答有没有幻觉：

```python
def check_hallucination(question, answer, context):
    """用裁判模型检查回答是否忠于参考资料"""
    judge_prompt = f"""你是事实核查员。请判断【回答】是否完全基于【参考资料】，有没有编造。

【参考资料】
{context}

【问题】
{question}

【回答】
{answer}

请检查：
1. 回答中的每个事实是否都能在参考资料中找到依据？
2. 有没有参考资料中没提到、却出现在回答里的信息？

输出 JSON：
{{"忠实": true/false, "可疑内容": ["..."], "理由": "..."}}
"""
    result = client.messages.create(
        model="claude-opus-4-5",
        temperature=0,
        messages=[{"role": "user", "content": judge_prompt}]
    )
    return result  # 解析 JSON
```

这个"忠实度（Faithfulness）"检查，是 RAG 系统里最重要的幻觉指标。

---

### 方法2：自洽性检查（Self-Consistency）

让模型对同一个问题答多次，如果每次答得都不一样，说明它在瞎猜：

```python
def self_consistency_check(question, n=5):
    answers = []
    for _ in range(n):
        resp = client.messages.create(
            model="claude-sonnet-4-6",
            temperature=0.7,   # 故意调高，看它稳不稳
            messages=[{"role": "user", "content": question}]
        )
        answers.append(extract_answer(resp))

    # 如果5次答案高度一致 → 可信
    # 如果5次答案五花八门 → 很可能是幻觉
    return measure_agreement(answers)
```

原理：**它真知道的事，怎么问都一样；它在编的事，每次编得都不同。**

---

### 方法3：让模型自我反思（Self-Reflection）

生成后，让模型回头审视自己的回答：

```python
reflection_prompt = f"""这是你刚才的回答：
{answer}

请重新审视：其中有没有你其实不太确定、或可能是猜测的内容？
如果有，请指出并修正。
"""
```

模型在"挑自己毛病"模式下，往往能发现刚才的过度自信。

---

### 方法4：用规则/工具硬核对

对于能客观验证的内容，别用模型验，用代码验：

```python
# 模型说某个 API 存在 → 真的去调一下
# 模型给了个数学计算 → 真的用 Python 算一遍
# 模型引用了某条数据 → 真的去数据库查一下

def verify_citation(answer, knowledge_base):
    cited_facts = extract_claims(answer)
    for fact in cited_facts:
        if not knowledge_base.contains(fact):
            flag_as_hallucination(fact)
```

能用确定性手段验证的，永远别用概率性手段（模型）验证。

## 第六部分：第5层 —— 兜底机制（分）

当系统判断"这个回答可能有幻觉"或"模型不确定"时，要有安全的降级路径。

### 策略1：置信度门槛 + 拒答

```python
def safe_answer(question):
    answer, confidence = generate_with_confidence(question)

    if confidence < 0.6:
        return "这个问题我没有足够把握，建议你咨询专业人士或查阅权威来源。"
    return answer
```

宁可拒答，不可错答——尤其在医疗、法律、金融等高风险场景。

---

### 策略2：人工介入（Human-in-the-loop）

高风险回答，先给人看，人确认了再发出去：

```
模型生成 → 置信度低/涉及关键决策 → 转人工审核 → 人确认 → 发出
```

---

### 策略3：透明地暴露不确定性

不要让用户误以为模型 100% 确定。在 UI 上明确标注：

```
回答：根据现有资料，可能是 XXX。
⚠️ 此回答由 AI 生成，可能存在误差，请核实重要信息。
[查看引用来源]
```

让用户保持警惕，本身就是一种防御。

## 第七部分：综合实战 —— 一个抗幻觉的问答函数（总）

把五层防御串起来，看一个完整的例子：

```python
from anthropic import Anthropic

client = Anthropic()

def anti_hallucination_qa(question: str, knowledge_base) -> dict:
    """一个综合了五层防御的问答函数"""

    # ===== 第1层：RAG 检索真实知识 =====
    contexts = knowledge_base.search(question, top_k=5)
    contexts = rerank(question, contexts)          # 重排序，保证相关
    contexts = [c for c in contexts if c.score > 0.7]  # 阈值过滤噪音

    if not contexts:
        # 第5层兜底：没检索到资料，直接拒答
        return {
            "answer": "根据现有资料库，我没有找到相关信息，无法回答。",
            "confidence": "none",
            "sources": []
        }

    context_text = "\n\n".join(
        f"[来源{i+1}] {c.text}" for i, c in enumerate(contexts)
    )

    # ===== 第2层 + 第3层：提示约束 + 生成控制 =====
    prompt = f"""请严格根据以下参考资料回答问题。

【参考资料】
{context_text}

【问题】
{question}

要求：
1. 只能使用参考资料中的信息，不要使用你自己的知识。
2. 每个论断后用 [来源N] 标注依据。
3. 如果参考资料无法回答，请直接说"根据现有资料无法回答"，不要编造。
"""

    response = client.messages.create(
        model="claude-sonnet-4-6",
        temperature=0,                # 第3层：最确定的生成
        max_tokens=1024,
        messages=[{"role": "user", "content": prompt}]
    )
    answer = response.content[0].text

    # ===== 第4层：事后验证忠实度 =====
    faithfulness = check_hallucination(question, answer, context_text)

    if not faithfulness["忠实"]:
        # 第5层兜底：验证不通过，降级
        return {
            "answer": "抱歉，我无法给出可靠的回答，建议查阅原始资料。",
            "confidence": "low",
            "sources": [c.source for c in contexts],
            "suspicious": faithfulness["可疑内容"]
        }

    return {
        "answer": answer,
        "confidence": "high",
        "sources": [c.source for c in contexts]
    }
```

每一层都对应着前面讲的一道防线。**没有任何单独一层能解决幻觉，但五层叠起来，幻觉率能从"经常发生"降到"罕见"。**

## 第八部分：常见误区与核心原则（总）

### 五个常见误区

| 误区 | 真相 |
|------|------|
| ❌ "换个更强的模型就不幻觉了" | 再强的模型也会幻觉，这是工程问题 |
| ❌ "加了 RAG 就不幻觉了" | 检索错了照样幻觉；RAG 只是降低，不是消除 |
| ❌ "temperature 调到 0 就没幻觉了" | 只能减少随机性，挡不住"模型本来就不知道" |
| ❌ "幻觉能被彻底根治" | 不能。只能控制到可接受范围 |
| ❌ "让模型自己保证不幻觉" | 模型无法可靠地自检，必须有外部验证 |

---

### 选择防御策略的决策建议

不是每个场景都要上全套五层，按风险和成本权衡：

| 场景 | 推荐防御层级 |
|------|------------|
| 闲聊、创意写作 | 几乎不用管（幻觉甚至是"创意"） |
| 一般知识问答 | 提示约束 + temperature=0 |
| 企业知识库问答 | RAG + 提示约束 + 引用来源 |
| 医疗/法律/金融 | 全部五层 + 人工审核 + 置信度拒答 |

---

### 一句话总结

> **解决幻觉的本质，是给一个"天生爱编故事"的系统，套上"必须看证据、必须留出处、不确定就闭嘴、有人帮它复核"的工程枷锁。**
>
> 你控制不了模型不去想象，但你能控制它把哪些话真正说出口。

---

## 本节重点

- 幻觉是 LLM 生成原理的**必然副产品**，无法根治，只能控制
- 解决思路是**五层防御**：RAG 注入知识 → 提示约束 → 生成控制 → 事后验证 → 兜底降级
- **RAG 是主力**，但检索质量决定了幻觉控制的上限
- 提示层面，**显式授权"可以说不知道"**是性价比最高的一招
- 事后用**第二个模型做忠实度核查**（LLM-as-a-Judge）是 RAG 系统的标配
- 高风险场景必须有**拒答兜底**和**人工介入**
