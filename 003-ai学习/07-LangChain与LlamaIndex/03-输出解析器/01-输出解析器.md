# 01 输出解析器

## 学习目标

这一节学习 Output Parser。

你需要掌握：

- 为什么需要输出解析。
- 字符串输出和结构化输出的区别。
- JSON 输出为什么容易不稳定。
- 如何用 Pydantic 约束模型输出。

## 为什么需要输出解析器

大模型默认输出自然语言。

例如：

```text
这个用户的情绪是积极的。
```

但程序更希望拿到结构化数据：

```json
{
  "label": "positive",
  "score": 0.92
}
```

输出解析器的作用就是把模型输出变成程序可处理的数据。

## 常见输出类型

### 字符串

适合：

- 普通聊天
- 文案生成
- 总结
- 翻译

### JSON

适合：

- 分类结果
- 抽取字段
- 配置生成
- 测试用例生成

### Pydantic 模型

适合：

- 需要字段类型校验
- 需要必填字段
- 需要嵌套结构
- 需要更稳定的接口返回

## 典型流程

```text
Prompt
    ↓
Model
    ↓
Output Parser
    ↓
Python dict / Pydantic object
```

常见组合：

```python
chain = prompt | model | parser
```

## Pydantic 输出示例

假设要让模型提取用户意图：

```python
from pydantic import BaseModel, Field


class IntentResult(BaseModel):
    intent: str = Field(description="用户意图")
    confidence: float = Field(description="置信度，0 到 1")
```

期望输出：

```json
{
  "intent": "查询订单",
  "confidence": 0.87
}
```

## 输出解析测试重点

测试输出解析时要关注：

- 是否是合法 JSON
- 字段是否齐全
- 字段类型是否正确
- 枚举值是否符合预期
- 错误输入时是否能稳定返回
- 模型是否夹带多余解释文本

## 建议练习

1. 让模型输出纯文本总结。
2. 让模型输出 JSON。
3. 定义一个 Pydantic 模型。
4. 让模型按 Pydantic 字段输出。
5. 故意输入复杂文本，观察输出是否稳定。

## 本节重点

- AI 应用不能只依赖自然语言输出。
- 输出解析器让模型输出更适合程序处理。
- JSON 输出要做校验。
- Pydantic 适合约束结构化输出。
- 输出解析是 AI 接口自动化测试的重要基础。
