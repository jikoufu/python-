# AI 测试学习路径

## 阶段 1：理解 AI 测试的特殊性

目标：知道为什么 AI 应用不能只用传统测试方法。

学习内容：

- AI 测试是什么
- 非确定性输出
- 幻觉问题
- 指令遵循
- 质量评估
- 安全风险

阶段成果：

- 能解释 AI 测试和传统测试的区别。
- 能列出 AI 应用常见风险。
- 能理解为什么需要测试集和评估指标。

## 阶段 2：测试大模型应用的基础能力

目标：能测试一个普通聊天或文本生成接口。

学习内容：

- 单轮问答测试
- 多轮对话测试
- 分类任务测试
- 信息抽取测试
- 延迟和成本测试

建议练习：

1. 准备 10 条问题。
2. 调用 AI 应用接口。
3. 保存每次回答。
4. 人工判断回答是否合格。
5. 记录失败样本。

## 阶段 3：Prompt 测试

目标：能判断 Prompt 修改是否引入回归。

学习内容：

- Prompt 版本管理
- Prompt 回归测试
- 指令遵循测试
- 输出格式测试
- 边界输入测试

阶段成果：

- 能给 Prompt 建立测试样本。
- 能比较两个 Prompt 版本的效果。
- 能发现格式不稳定、拒答不合理、指令不遵循等问题。

## 阶段 4：RAG 测试

目标：能测试知识库问答是否可靠。

学习内容：

- 检索是否命中正确文档
- 回答是否基于上下文
- 回答是否有幻觉
- 引用是否准确
- 未知问题是否拒答

阶段成果：

- 能区分检索问题和生成问题。
- 能设计 RAG 测试样本。
- 能检查答案和知识库内容是否一致。

## 阶段 5：Agent 测试

目标：能测试带工具调用和多步骤推理的 AI Agent。

学习内容：

- 工具选择是否正确
- 工具参数是否正确
- 多步骤任务是否完成
- 中途失败是否能恢复
- 是否出现越权工具调用

阶段成果：

- 能为 Agent 编写任务型测试用例。
- 能检查每一步工具调用轨迹。
- 能发现工具调用错误和任务中断问题。

## 阶段 6：自动化评估

目标：能把 AI 测试变成可重复执行的自动化流程。

学习内容：

- 测试数据集
- 黄金答案
- 规则断言
- LLM-as-a-Judge
- 回归测试
- 测试报告

阶段成果：

- 能批量运行测试样本。
- 能输出通过率和失败原因。
- 能对 Prompt 或模型升级做回归验证。

## 阶段 7：安全与风险测试

目标：能测试 AI 应用是否存在安全风险。

学习内容：

- Prompt 注入
- 越权访问
- 敏感信息泄露
- 有害内容生成
- 数据污染
- 输出合规性

阶段成果：

- 能设计基础 AI 安全测试用例。
- 能发现常见 Prompt 注入风险。
- 能验证系统是否会泄露不该返回的信息。

## 阶段 8：实战项目

目标：完成一个 AI 问答系统测试框架。

项目功能：

- 读取测试样本
- 调用 AI 应用接口
- 保存响应结果
- 自动检查格式
- 人工或模型辅助打分
- 输出 Markdown 测试报告

推荐数据格式：

```json
{
  "id": "case_001",
  "question": "FastAPI 如何定义 GET 接口？",
  "expected_keywords": ["@app.get", "路径", "函数"],
  "category": "基础问答"
}
```

最终你应该能回答：

- AI 应用怎么测？
- Prompt 怎么做回归？
- RAG 怎么判断有没有幻觉？
- Agent 怎么测试工具调用？
- 如何把 AI 测试自动化？

## 阶段 9：Selenium 自动化测试

目标：能用浏览器自动化测试 AI 应用前端页面。

学习内容：

- Selenium 基础
- 浏览器启动和关闭
- 元素定位
- 点击和输入
- 显式等待
- pytest 集成
- 页面对象模式
- AI 聊天页面测试

阶段成果：

- 能写一个打开网页的 Selenium 脚本。
- 能定位输入框和按钮。
- 能模拟用户提问并等待 AI 回复。
- 能对页面文本做断言。
- 能把 Selenium 用例接入 pytest。

## 阶段 10：pytest 接口自动化测试

目标：能用 pytest 测试普通接口和 AI 应用接口。

学习内容：

- pytest 基础
- requests 请求接口
- 状态码断言
- JSON 断言
- fixture
- 参数化
- FastAPI TestClient
- AI 回答质量断言
- 测试报告

阶段成果：

- 能写 GET、POST 接口自动化用例。
- 能使用 fixture 管理登录 token 和 base_url。
- 能批量测试多组接口数据。
- 能测试 AI 接口的格式、内容和耗时。
- 能生成基础测试报告。
