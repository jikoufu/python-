## 阶段 1：Docker

目标：能用 Docker 启动任意中间件，能为 Python 应用构建镜像并用 docker-compose 编排。

### 核心概念

```text
镜像（Image）：只读模板，类似安装包
容器（Container）：镜像运行后的实例，类似进程
仓库（Registry）：存放镜像的地方，默认是 Docker Hub
```

### 安装

```bash
# Ubuntu
curl -fsSL https://get.docker.com | sh
sudo usermod -aG docker $USER
# 重新登录后生效

# 验证
docker run hello-world
```

### 常用命令

```bash
# 镜像操作
docker pull nginx:1.25               # pull：拉取镜像
docker images                        # images：列出本地镜像
docker rmi nginx:1.25                # rmi = remove image，删除镜像

# 容器操作
docker run -d -p 8080:80 --name web nginx   # run：创建并启动容器；-d = detached，后台运行；-p = publish，映射端口
docker ps                                    # ps = process status，查看运行中容器
docker ps -a                                 # -a = all，查看所有容器
docker stop web                              # 停止容器
docker rm web                                # rm = remove，删除容器
docker logs web -f                           # -f = follow，实时跟踪日志
docker exec -it web bash                     # exec：执行命令；-it = interactive + tty，交互式终端

# 数据卷
docker run -v /host/path:/container/path nginx  # -v = volume，挂载目录
```

### 索引原理

```sql
-- 查看查询执行计划（EXPLAIN）
EXPLAIN SELECT * FROM articles WHERE user_id = 1 AND status = 'published';

-- 结果中关注：
-- type: ALL（全表扫描，慢）→ ref/range（索引扫描，快）
-- key: 实际使用的索引
-- rows: 扫描的行数

-- 索引选择原则
-- 1. 高频查询的 WHERE 条件字段
-- 2. 联合索引遵循最左前缀原则
-- 3. 区分度低的字段不加索引（如性别）
-- 4. 避免在索引列上做函数运算
```

### 事务与 ACID

```sql
-- 事务示例：转账
START TRANSACTION;

UPDATE accounts SET balance = balance - 100 WHERE id = 1;
UPDATE accounts SET balance = balance + 100 WHERE id = 2;

-- 检查无误后提交
COMMIT;

-- 发现问题时回滚
ROLLBACK;
```

| 特性 | 说明 |
|------|------|
| Atomicity 原子性 | 全成功或全失败 |
| Consistency 一致性 | 事务前后数据都合法 |
| Isolation 隔离性 | 事务之间互不干扰 |
| Durability 持久性 | 提交后数据不丢失 |

### 隔离级别与常见问题

```sql
-- 查看当前隔离级别
SELECT @@transaction_isolation;

-- 设置隔离级别
SET SESSION TRANSACTION ISOLATION LEVEL READ COMMITTED;
```

| 隔离级别 | 脏读 | 不可重复读 | 幻读 |
|---------|------|-----------|------|
| READ UNCOMMITTED | 有 | 有 | 有 |
| READ COMMITTED | 无 | 有 | 有 |
| REPEATABLE READ（MySQL 默认）| 无 | 无 | 无（MVCC 解决） |
| SERIALIZABLE | 无 | 无 | 无 |

### 缓存策略

**Cache-Aside（旁路缓存，最常用）**

```python
import redis.asyncio as aioredis
import json

redis = aioredis.from_url("redis://localhost:6379")

async def get_user(user_id: int):
    # 1. 先查缓存
    cache_key = f"user:{user_id}"
    cached = await redis.get(cache_key)
    if cached:
        return json.loads(cached)

    # 2. 缓存未命中，查数据库
    user = await db.get_user(user_id)
    if not user:
        return None

    # 3. 写入缓存，设置过期时间
    await redis.set(cache_key, json.dumps(user), ex=3600)
    return user

async def update_user(user_id: int, data: dict):
    await db.update_user(user_id, data)
    # 4. 更新数据库后删除缓存（下次读时重建）
    await redis.delete(f"user:{user_id}")
```

**常见缓存问题**

| 问题 | 现象 | 解决方案 |
|------|------|----------|
| 缓存穿透 | 查询不存在的数据，每次打到数据库 | 缓存空值 or 布隆过滤器 |
| 缓存雪崩 | 大量 key 同时过期，数据库被打爆 | 随机过期时间 + 多级缓存 |
| 缓存击穿 | 热点 key 过期，大量请求同时查 DB | 分布式锁 or 永不过期 |

### 分布式锁

```python
import uuid
import asyncio

async def acquire_lock(redis, resource: str, ttl: int = 10) -> str | None:
    lock_key = f"lock:{resource}"
    token = str(uuid.uuid4())
    # SET key value NX EX ttl（原子操作，不存在时才设置）
    result = await redis.set(lock_key, token, nx=True, ex=ttl)
    return token if result else None

async def release_lock(redis, resource: str, token: str):
    lock_key = f"lock:{resource}"
    # 用 Lua 脚本保证原子性（检查 + 删除）
    lua_script = """
    if redis.call('get', KEYS[1]) == ARGV[1] then
        return redis.call('del', KEYS[1])
    else
        return 0
    end
    """
    await redis.eval(lua_script, 1, lock_key, token)

# 使用
async def create_order(order_id: str):
    token = await acquire_lock(redis, f"order:{order_id}")
    if not token:
        raise Exception("获取锁失败，请稍后重试")
    try:
        # 执行业务逻辑
        pass
    finally:
        await release_lock(redis, f"order:{order_id}", token)
```

### 四种 Exchange 类型

| 类型 | 路由规则 | 适用场景 |
|------|----------|----------|
| Direct | routing_key 完全匹配 | 点对点消息 |
| Topic | routing_key 通配符匹配（`*` 一个词，`#` 多个词）| 分类路由 |
| Fanout | 广播到所有绑定队列 | 群发通知 |
| Headers | 按消息头匹配 | 少用 |

### Python 集成（aio-pika）

```python
import aio_pika
import json
import asyncio

# 生产者：发送任务
async def send_task(task_type: str, payload: dict):
    connection = await aio_pika.connect_robust("amqp://admin:password@localhost/")
    async with connection:
        channel = await connection.channel()
        queue = await channel.declare_queue("tasks", durable=True)

        await channel.default_exchange.publish(
            aio_pika.Message(
                body=json.dumps({"type": task_type, "data": payload}).encode(),
                delivery_mode=aio_pika.DeliveryMode.PERSISTENT,  # 消息持久化
            ),
            routing_key="tasks",
        )

# 消费者：处理任务
async def consume_tasks():
    connection = await aio_pika.connect_robust("amqp://admin:password@localhost/")
    channel = await connection.channel()
    await channel.set_qos(prefetch_count=10)    # 一次最多处理 10 条
    queue = await channel.declare_queue("tasks", durable=True)

    async with queue.iterator() as queue_iter:
        async for message in queue_iter:
            async with message.process():       # 自动 ack/nack
                try:
                    task = json.loads(message.body)
                    await handle_task(task)
                except Exception as e:
                    # 处理失败，消息会进入死信队列
                    raise

asyncio.run(consume_tasks())
```

### 消息可靠性保障

```text
生产者可靠性：
- Publisher Confirms：RabbitMQ 确认收到消息后才算成功

消息持久化：
- Queue durable=True：队列重启后不消失
- Message delivery_mode=PERSISTENT：消息重启后不丢失

消费者可靠性：
- 手动 ACK：消息处理成功后才确认，失败则 NACK 重新入队
- prefetch_count：限制未确认消息数，防止消费者过载
```

### 死信队列（处理失败消息）

```python
# 创建带死信队列的主队列
args = {
    "x-dead-letter-exchange": "dlx",      # 死信交换机
    "x-message-ttl": 30000,               # 消息 30 秒未消费进死信队列
}
main_queue = await channel.declare_queue("tasks", durable=True, arguments=args)

# 死信队列（存放处理失败的消息，人工排查）
dlq = await channel.declare_queue("tasks.dlq", durable=True)
```

## 阶段 6：Elasticsearch

目标：能建立全文搜索索引，实现中文分词搜索和基本的聚合分析。

### 核心概念

```text
Index（索引）≈ 数据库中的表
Document（文档）≈ 表中的一行（JSON 格式）

```

### 倒排索引原理

```text
普通数据库：文档 → 字段值（正向查找）
倒排索引：词项 → 包含该词的文档列表（反向查找）

示例：
文档1："Python 入门教程"
文档2："Python 高级编程"

倒排索引：
"python" → [文档1, 文档2]
"入门"   → [文档1]
"高级"   → [文档2]

搜索"python 高级"：取交集 → [文档2]
```

### Python 集成

```python
from elasticsearch import AsyncElasticsearch

es = AsyncElasticsearch("http://localhost:9200")

# 索引文档
async def index_article(article: dict):
    await es.index(
        index="articles",
        id=str(article["id"]),
        document=article
    )

# 搜索
async def search_articles(query: str, page: int = 1, size: int = 10):
    response = await es.search(
        index="articles",
        body={
            "query": {
                "multi_match": {
                    "query": query,
                    "fields": ["title^3", "content"]
                }
            },
            "from": (page - 1) * size,
            "size": size
        }
    )
    return {
        "total": response["hits"]["total"]["value"],
        "items": [hit["_source"] for hit in response["hits"]["hits"]]
    }

await es.close()
```
