# 05 数据卷与网络

## 学习目标

学完这一节，你需要掌握：

- 为什么需要数据卷。
- volume 和 bind mount 的区别。
- Docker 网络的作用。
- 容器之间如何通信。

## 为什么需要数据卷

容器删除后，容器内部数据默认会丢失。

比如 MySQL 容器里有数据库数据。

如果没有数据卷：

```text
删除容器 = 数据可能丢失
```

所以中间件通常必须挂载数据卷。

## volume

Docker 管理的数据卷。

创建：

```bash
docker volume create mysql_data
```

查看：

```bash
docker volume ls
```

使用：

```bash
docker run -d --name mysql -v mysql_data:/var/lib/mysql mysql:8.0
```

优点：

- Docker 管理。
- 跨容器复用。
- 适合数据库数据。

## bind mount

把宿主机目录挂载到容器。

示例：

```bash
docker run -d --name nginx -v D:/nginx/html:/usr/share/nginx/html nginx
```

优点：

- 文件在宿主机上可直接编辑。
- 适合配置文件、静态资源、开发代码。

缺点：

- 路径依赖宿主机。
- Windows 路径容易写错。

## volume vs bind mount

| 类型 | 适合场景 |
|------|----------|
| volume | 数据库数据、Redis 数据 |
| bind mount | 配置文件、代码、静态资源 |

## Docker 网络

容器之间通信依赖 Docker 网络。

查看网络：

```bash
docker network ls
```

创建网络：

```bash
docker network create backend
```

启动容器加入网络：

```bash
docker run -d --name redis --network backend redis:7
docker run -d --name app --network backend myapp:v1
```

app 可以通过容器名访问 redis：

```text
redis:6379
```

## 端口映射

容器内部端口不等于宿主机端口。

示例：

```bash
docker run -d -p 8080:80 nginx
```

含义：

```text
宿主机 8080 → 容器 80
```

访问：

```text
http://127.0.0.1:8080
```

## 常见错误

### 1. 容器连接 MySQL 写 localhost

错误：

```text
mysql://root:123456@127.0.0.1:3306/app_db
```

如果 app 在容器里，这个地址指 app 容器自己。

compose 中应该写：

```text
mysql://root:123456@db:3306/app_db
```

### 2. 删除 volume 导致数据丢失

```bash
docker compose down -v
```

会删除数据卷。

生产或重要数据环境不要随便执行。

### 3. 端口被占用

如果宿主机 3306 已经被占用：

```bash
docker run -p 3306:3306 mysql:8.0
```

会失败。

可以换端口：

```bash
docker run -p 3307:3306 mysql:8.0
```

访问宿主机：

```text
127.0.0.1:3307
```

容器内部仍然是：

```text
3306
```

## 本节重点

- 数据库必须挂载数据卷。
- volume 适合持久化数据。
- bind mount 适合挂载配置和代码。
- 容器之间用服务名或容器名通信。
- 容器里的 localhost 是容器自己。
