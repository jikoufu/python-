## 学习原则

- 先理解题型模板，再刷题。
- 每道题先自己写暴力思路，再看优化思路。
- 代码写完后必须手动走一遍样例。
- 面试时重点讲清楚：为什么这么做、边界条件是什么、复杂度是多少。
