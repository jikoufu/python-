# FastAPI 学习路径步骤

## 阶段 1：准备环境

目标：能启动一个最简单的 FastAPI 服务。

### 学习内容

- Python 虚拟环境
- FastAPI 安装
- Uvicorn 启动服务
- Swagger UI 自动文档

### 建议步骤

1. 创建或激活虚拟环境。
2. 安装依赖：

```bash
pip install fastapi uvicorn
```

3. 创建 `main.py`。
4. 写第一个接口：

```python
from fastapi import FastAPI

app = FastAPI()


@app.get("/")
def read_root():
    return {"message": "Hello FastAPI"}
```

5. 启动服务：

```bash
uvicorn main:app --reload
```

6. 打开接口文档：

```text
http://127.0.0.1:8000/docs
```

### 阶段成果

- 能启动 FastAPI 服务。
- 能访问 `/docs`。
- 能写一个 GET 接口。

## 阶段 2：掌握接口参数

目标：能写常见的 GET、POST、PUT、DELETE 接口。

### 学习内容

- 路径参数
- 查询参数
- 请求体
- Header
- Cookie
- 状态码

### 建议练习

1. 写一个根据用户 ID 查询用户的接口。
2. 写一个分页查询接口。
3. 写一个新增用户接口。
4. 写一个修改用户接口。
5. 写一个删除用户接口。

### 示例方向

```text
GET    /users/{user_id}
GET    /users?page=1&page_size=10
POST   /users
PUT    /users/{user_id}
DELETE /users/{user_id}
```

### 阶段成果

- 理解不同参数的来源。
- 能根据业务场景选择合适的 HTTP 方法。
- 能返回合适的状态码。

## 阶段 3：学习 Pydantic 数据校验

目标：能使用模型约束请求和响应数据。

### 学习内容

- BaseModel
- 字段类型
- 默认值
- 可选字段
- 字段校验
- 响应模型

### 建议练习

1. 定义 `UserCreate` 模型。
2. 定义 `UserUpdate` 模型。
3. 定义 `UserResponse` 模型。
4. 给邮箱、年龄、密码增加校验规则。
5. 区分请求模型和响应模型。

### 阶段成果

- 能用 Pydantic 校验请求体。
- 能避免把密码等敏感字段返回给前端。
- 能让接口文档自动显示数据结构。

## 阶段 4：掌握依赖注入

目标：能把公共逻辑抽离出来复用。

### 学习内容

- Depends
- 公共查询参数
- 登录用户依赖
- 数据库连接依赖
- 多层依赖

### 建议练习

1. 封装分页参数依赖。
2. 封装当前用户依赖。
3. 封装数据库 Session 依赖。
4. 给需要登录的接口增加依赖。

### 阶段成果

- 理解 FastAPI 依赖注入的作用。
- 能减少接口里的重复代码。
- 为数据库和认证打基础。

## 阶段 5：接入数据库

目标：能把数据保存到数据库，而不是只存在内存里。

### 学习内容

- SQLAlchemy
- 数据库连接
- ORM 模型
- Session
- CRUD
- Alembic 迁移

### 建议练习

1. 连接 SQLite 或 MySQL。
2. 创建用户表。
3. 实现用户新增、查询、修改、删除。
4. 使用 Alembic 管理表结构变化。
5. 把数据库 Session 改造成依赖注入。

### 阶段成果

- 能完成数据库增删改查。
- 能理解 ORM 模型和 Pydantic 模型的区别。
- 能使用迁移工具维护表结构。

## 阶段 6：实现认证与权限

目标：能实现注册、登录和接口保护。

### 学习内容

- 密码哈希
- OAuth2PasswordBearer
- JWT
- 当前登录用户
- 权限控制

### 建议练习

1. 实现用户注册。
2. 保存加密后的密码。
3. 实现登录接口。
4. 登录成功后返回 JWT。
5. 解析 JWT 获取当前用户。
6. 保护需要登录的接口。
7. 增加管理员权限判断。

### 阶段成果

- 能完成基本登录流程。
- 能保护接口不被匿名访问。
- 能区分普通用户和管理员。

## 阶段 7：整理项目结构

目标：把代码从单文件整理成真实项目结构。

### 推荐结构

```text
app/
├── main.py
├── core/
│   ├── config.py
│   └── security.py
├── api/
│   └── v1/
│       ├── router.py
│       └── endpoints/
├── models/
├── schemas/
├── crud/
├── db/
└── services/
```

### 学习内容

- Router 拆分
- 配置管理
- 分层思想
- 统一响应
- 全局异常
- 日志管理

### 阶段成果

- 项目结构清晰。
- 接口、模型、数据库、业务逻辑分层明确。
- 后续功能更容易扩展。

## 阶段 8：测试和部署

目标：能验证接口正确性，并部署到服务器。

### 学习内容

- pytest
- TestClient
- 测试数据库
- Docker
- docker-compose
- Gunicorn
- Nginx

### 建议练习

1. 给用户注册接口写测试。
2. 给登录接口写测试。
3. 给需要登录的接口写测试。
4. 编写 Dockerfile。
5. 使用 docker-compose 启动应用和数据库。
6. 部署到 Linux 服务器。

### 阶段成果

- 能写接口自动化测试。
- 能使用 Docker 启动项目。
- 能把 FastAPI 项目部署上线。

## 最终实战项目

建议完成一个博客 API 项目。

### 功能清单

- 用户注册
- 用户登录
- JWT 认证
- 用户信息查询
- 文章发布
- 文章列表
- 文章详情
- 文章修改
- 文章删除
- 管理员权限
- 接口测试
- Docker 部署

### 推荐接口

```text
POST   /api/v1/auth/register
POST   /api/v1/auth/login
GET    /api/v1/users/me
POST   /api/v1/articles
GET    /api/v1/articles
GET    /api/v1/articles/{article_id}
PUT    /api/v1/articles/{article_id}
DELETE /api/v1/articles/{article_id}
```

## 学习建议

- 不要只看文档，一定要多写接口。
- 每学完一个阶段，都把代码整理进小项目。
- 遇到概念不清楚时，优先写最小示例验证。
- 数据库、认证、依赖注入是 FastAPI 后端开发的重点。
- 最后一定要补测试和部署，否则项目不算完整。
