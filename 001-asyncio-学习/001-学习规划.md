# 二、阶段1：基础认知（能跑）

## 🎯 目标

理解 async/await，不再“看不懂代码”。

## 必会知识

* `async def`
* `await`
* `asyncio.run()`
* `asyncio.sleep()`

## 核心理解

你必须能回答：

> async 函数和普通函数区别是什么？

标准答案（你要记住）：

* async 函数返回的是 **协程对象**
* 需要事件循环调度才会执行
* 协程对象本质是：一个“还没执行的任务描述”
---
    
## 最小代码

```python
import asyncio

async def main():
    print("start")
    await asyncio.sleep(1)
    print("end")

asyncio.run(main())
```

---

## ✅ 验收标准

你必须能解释：

* 为什么必须用 `asyncio.run`
* 为什么要 `await`
* 为什么 `sleep` 不用 `time.sleep`

---

## 🧪 练习

写 3 个 async 函数，每个 sleep 不同时间，顺序执行。

# 三、阶段2：并发模型（能并发）

## 🎯 目标

理解“为什么 asyncio 会快”。

## 必会知识

* `asyncio.gather()`
* 并发 vs 串行
* I/O vs CPU 区别

---

## 核心理解（非常关键）

> asyncio 并发的本质是什么？

标准答案：

**任务在 await 时让出执行权，事件循环调度其他任务执行。**

---

## 对比代码（必须掌握）

### 串行

```python
await task1()
await task2()
```

### 并发

```python
await asyncio.gather(task1(), task2())
```

---

## ✅ 验收标准

你必须能解释：

* 为什么 3 个 2 秒任务可以 2 秒完成
* 为什么不是多线程

---

## 🧪 练习

模拟 10 个接口请求（sleep），统计耗时对比：

* 串行
* gather 并发

# 四、阶段3：任务控制（能控场）

## 🎯 目标

你开始“像工程师一样用 asyncio”。

---

## 必会知识

* `asyncio.create_task()`
* Task vs coroutine
* `await task`
* 并发执行 vs 顺序等待

---

## 核心区别（重点）

| 写法              | 含义      |
| --------------- | ------- |
| `await task()`  | 立即执行并等待 |
| `create_task()` | 后台执行    |

---

## 示例

```python
task = asyncio.create_task(work())

# 不阻塞
print("继续执行")

await task
```

---

## ✅ 验收标准

你必须能回答：

* create_task 和 await 的区别
* 为什么 create_task 可以“并发推进”

---

## 🧪 练习

写一个：

* 3 个后台任务
* 主线程继续执行
* 最后统一收结果

# 五、阶段4：真实应用（能落地）

## 🎯 目标

能用在你的自动化测试 / AI 测试平台里

（这一阶段和你当前工作强相关）

---

## 必会知识

### 1）并发请求接口（重点）

配合：

* aiohttp

```python
import aiohttp
import asyncio

async def fetch(session, url):
    async with session.get(url) as resp:
        return await resp.text()

async def main():
    async with aiohttp.ClientSession() as session:
        results = await asyncio.gather(
            fetch(session, "http://example.com"),
            fetch(session, "http://example.com"),
        )
```

---

### 2）超时控制

```python
await asyncio.wait_for(task(), timeout=3)
```

---

### 3）异常处理

```python
results = await asyncio.gather(task1(), task2(), return_exceptions=True)
```

---

### 4）限流（非常重要）

```python
sem = asyncio.Semaphore(5)

async with sem:
    await task()
```

👉 这个你做接口压测/批量调用一定会用到

---

## ✅ 验收标准

你必须能实现：

* 并发 100 个接口请求

* 有失败不崩
* 有超时机制
