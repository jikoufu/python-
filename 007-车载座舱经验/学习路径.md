# 车载座舱经验学习路径

## 阶段 1：理解智能座舱是什么

目标：先建立整体认知。

学习内容：

- 汽车五域架构
- 动力域
- 底盘域
- 车身域
- 座舱域
- 智驾域
- 智能座舱定义
- IVI 中控屏
- 仪表屏
- HUD
- 后排娱乐屏
- 座舱域控制器
- 车机系统

阶段成果：

- 能画出座舱系统大致组成。
- 能说清楚汽车五域架构。
- 能说明座舱域和其他域的信号联动关系。
- 能说清楚中控、仪表、HUD 的区别。
- 能知道车载座舱和普通 App 的不同。

## 阶段 2：学习 HMI 与交互体验

目标：理解座舱页面和交互测试重点。

学习内容：

- HMI 页面结构
- 页面跳转
- 菜单层级
- 弹窗
- Toast
- 触控响应
- 夜间模式
- 多屏联动

阶段成果：

- 能设计座舱 UI 测试用例。
- 能发现交互不一致、页面遮挡、响应慢等问题。

## 阶段 3：学习语音交互

目标：掌握座舱语音助手测试思路。

学习内容：

- 唤醒
- ASR
- NLU
- TTS
- 可见即可说
- 连续对话
- 语音车控
- 噪声环境测试

阶段成果：

- 能设计语音唤醒、识别、理解、播报测试用例。
- 能区分是 ASR 错、NLU 错，还是执行错。

## 阶段 4：学习导航与地图

目标：掌握导航功能测试和联动测试。

学习内容：

- 搜索目的地
- 路线规划
- 导航开始/暂停/结束
- 偏航重算
- 实时路况
- 语音播报
- 仪表/HUD 联动

阶段成果：

- 能测试导航主流程。
- 能设计异常路线、无网、定位漂移等场景。

## 阶段 5：学习多媒体与蓝牙

目标：掌握座舱高频娱乐功能测试。

学习内容：

- 音乐播放
- 视频播放
- 本地媒体
- 在线媒体
- 蓝牙配对
- 蓝牙电话
- 蓝牙音乐
- 音源切换

阶段成果：

- 能测试蓝牙连接、断连、重连。
- 能测试播放、暂停、切歌、音量和焦点抢占。

## 阶段 6：学习车控与车辆信号

目标：理解座舱如何控制车辆功能。

学习内容：

- 空调
- 座椅
- 车窗
- 车灯
- 驾驶模式
- 车辆状态
- CAN 信号
- 车控安全

阶段成果：

- 能设计车控功能测试用例。
- 能关注权限、安全、状态一致性。

## 阶段 7：学习车机通信与接口

目标：理解车机和其他系统怎么交互。

学习内容：

- 车机和云端
- 车机和 TBOX
- 车机和座舱域控
- HTTP
- MQTT
- CAN
- SOME/IP
- OTA

阶段成果：

- 能看懂基本接口链路。
- 能区分本地问题、云端问题、通信问题。

## 阶段 8：学习诊断与日志分析

目标：具备问题定位能力。

学习内容：

- Android logcat
- 应用日志
- 系统日志
- 崩溃日志
- 网络抓包
- DTC
- 复现路径

阶段成果：

- 能收集日志。
- 能根据日志初步判断问题模块。
- 能写清楚缺陷复现步骤和证据。

## 阶段 9：沉淀座舱测试经验

目标：把知识转成测试能力。

学习内容：

- 功能测试
- 兼容性测试
- 稳定性测试
- 性能测试
- 异常测试
- 安全测试
- 用户体验测试

阶段成果：

- 能为一个座舱模块写完整测试方案。
- 能输出测试报告。

## 阶段 10：沉淀项目经验

目标：形成可复用、可面试表达的项目经验。

学习内容：

- 项目背景
- 负责模块
- 测试范围
- 测试策略
- 典型缺陷
- 风险控制
- 上线复盘

阶段成果：

- 能写出一段简历项目经验。
- 能讲清楚自己解决过的复杂问题。

## 阶段 11：准备面试

目标：把经验整理成表达能力。

重点准备：

- 智能座舱是什么
- 语音模块怎么测
- 导航模块怎么测
- 蓝牙问题怎么排查
- 车控测试关注什么
- 日志如何定位问题
- 项目中遇到过什么复杂缺陷

最终成果：

- 一套座舱测试用例模板
- 一份座舱测试方案
- 一份座舱项目经验总结
- 一份面试题整理

## 阶段 12：学习 CANoe 工具

目标：掌握座舱测试中常用的车辆信号分析和仿真能力。

学习内容：

- CANoe 是什么
- CAN 基础
- ECU
- 报文和信号
- DBC
- Trace Window
- Data Window
- Panel
- 信号仿真
- 座舱车控测试场景
- CANoe 面试表达

阶段成果：

- 能看懂 CANoe 中的信号变化。
- 能通过 CANoe 模拟车速、档位、车门、安全带等信号。
- 能验证车机 UI 和车辆信号是否一致。
- 能在面试中讲清楚 CANoe 的使用场景。
