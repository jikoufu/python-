# 03 LIN 与车载以太网

## 学习目标

学完这一节，你需要能：

- 说清 LIN 是什么、用在哪、和 CAN 什么关系（了解级）
- 讲明白座舱为什么大量用以太网
- 说清车载以太网和普通以太网的区别
- 理解 SOME/IP 的核心概念（服务发现、方法调用、事件订阅）
- 知道 DoIP、TSN 是干什么的
- 理解网关在整车通信里的角色

---

## 第一部分：LIN（轻量了解）

### LIN 是什么

LIN（Local Interconnect Network，本地互连网络）是一种**比 CAN 更便宜、更慢**的总线。

### 一句话定位

> **LIN 是 CAN 的低成本补充。**

CAN 又快又可靠但贵，有些功能根本不需要那么快，用 CAN 就浪费。这时候用 LIN。

### LIN 的特点

```text
- 主从架构：一个主节点（Master）+ 多个从节点（Slave）
- 主节点说了算：什么时候谁能说话，由主节点调度
- 单线通信：一根线（比 CAN 的双绞线还省）
- 速率低：最高约 20 kbps（CAN 是 1 Mbps）
```

### 用在哪

低速、低成本、对实时性要求不高的场景：

```text
车窗、座椅调节、灯光、后视镜、雨刮、空调风门 ...
```

### LIN 帧结构（知道即可）

```text
帧头(主节点发) ┌ 同步间隔
               ├ 同步场
               └ PID（受保护的标识符）
帧响应(从节点发)┌ 数据场
               └ 校验和
```

> 座舱测试岗位 LIN 是"了解"级别。能说出"LIN 是主从架构、低速低成本、用于车窗座椅这类场景、是 CAN 的补充"就够了。

### CAN vs LIN 速记

| 维度 | CAN | LIN |
|------|-----|-----|
| 架构 | 多主，平等 | 主从 |
| 速率 | 高（1 Mbps） | 低（20 kbps） |
| 成本 | 较高 | 低 |
| 线 | 双绞线两根 | 单线 |
| 场景 | 动力、底盘等关键功能 | 车窗、座椅等舒适功能 |

## 第二部分：车载以太网（座舱重点）

这块对智能座舱比 LIN 重要得多，**座舱是以太网重灾区**。

### 为什么座舱/智驾要用以太网

CAN 一帧才 8 字节（CAN FD 64 字节），传传控制信号够，但座舱要传的是：

```text
- 摄像头图像（360 环视、行车记录）
- 高清音视频
- 高精地图数据
- 域控制器之间的大量数据
```

这些是**大带宽数据**，CAN 根本扛不住。以太网带宽大（100Mbps / 1Gbps 起步），正合适。

### 车载以太网 vs 普通以太网

最大区别在**物理层**：

```text
普通以太网：     网线 8 芯（4 对双绞线），水晶头
车载以太网：     单对双绞线（就 2 根线！）
                标准：100BASE-T1（百兆）、1000BASE-T1（千兆）
```

为什么用单对线？**为了省线束重量和成本**——车上线越少越好。

```text
记忆点：100BASE-T1 / 1000BASE-T1 中的 "T1" = 单对双绞线（Twisted pair, 1 pair）
```

其他方面（IP、TCP/UDP 这些协议栈）和普通以太网基本一致。

## 第三部分：SOME/IP（理解核心概念）

### SOME/IP 是什么

SOME/IP（Scalable service-Oriented MiddlewarE over IP）是车载以太网上**面向服务**的通信协议。

### "面向服务"是什么意思

传统 CAN 是"面向信号"——我不停广播"车速=60"，不管有没有人要。

SOME/IP 是"面向服务"——像点菜：

```text
面向信号（CAN）：   厨房不停把所有菜端出来，谁要谁拿
面向服务（SOME/IP）：你要什么，我给你什么
```

### 三个核心概念（面试常问）

| 概念 | 通俗解释 |
|------|---------|
| **服务发现（SD）** | "谁能提供导航数据服务？" 节点上线时广播自己能提供什么服务，别人来发现 |
| **方法调用（Method）** | 像调函数：客户端请求 → 服务端处理 → 返回结果（一问一答） |
| **事件订阅（Event）** | "我订阅车速变化，你一变就推给我" 订阅后服务端主动推送 |

> 记住这三个词：**服务发现、方法调用、事件订阅**。面试问 SOME/IP 答这三点准没错。

## 第四部分：DoIP、TSN、网关

### DoIP（了解）

DoIP（Diagnostics over IP）= **基于以太网的诊断**。

传统诊断走 CAN（速度慢），现在很多诊断走以太网（DoIP），刷写、读故障码快得多。知道"DoIP 是以太网上的诊断"即可。

### TSN（知道名词即可）

TSN（Time-Sensitive Networking，时间敏感网络）= **让以太网也能保证实时性**。

普通以太网传输有延迟、不确定，但车上有些数据（如音视频同步）要求"准时到达"。TSN 给以太网加了一套时间保证机制。

> 知道"TSN 是给以太网做实时性保证的"就够了。

### 网关（Gateway）

整车有 CAN、LIN、以太网多种总线，它们**说不同的"语言"**，需要一个翻译+转发的中枢——这就是网关。

```text
        ┌─── CAN（动力域）
        │
网关 ───┼─── LIN（车身舒适）
        │
        └─── 以太网（座舱/智驾骨干）

作用：在不同总线之间转换协议、路由数据
```

举例：刹车信号在动力域 CAN 上，座舱要显示"正在刹车"图标，数据就要**经网关从 CAN 转到座舱以太网**。

> 整车数据流转的"立交桥"就是网关。座舱很多信号其实来自其他域，都是网关转过来的。

## 第五部分：把整条链路串起来

面试时能把这条链路讲顺，很加分：

```text
传感器/其他域 ECU
     ↓ （域内 CAN/LIN）
域控制器
     ↓ （以太网骨干）
网关（跨域转换路由）
     ↓
座舱域控制器
     ↓
中控屏/仪表显示给用户
```

一句话：**域内用 CAN/LIN，域间用以太网骨干，网关负责跨域翻译路由。**

---

## 本节重点

- LIN：**主从架构、低速低成本、车窗座椅等场景、是 CAN 的补充**（了解级）
- 座舱大量用以太网，因为要传**图像/音视频/地图等大带宽数据**
- 车载以太网用**单对双绞线**，标准 100BASE-T1 / 1000BASE-T1
- SOME/IP 是面向服务通信，三核心：**服务发现、方法调用、事件订阅**
- DoIP = 以太网诊断；TSN = 以太网实时性保证
- 网关在 CAN/LIN/以太网之间转换路由，是整车数据"立交桥"
