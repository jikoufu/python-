# 03 HMI 常见问题与经验

## 学习目标

这一节用于沉淀 HMI 测试中的常见问题和经验。

你需要掌握：

- HMI 常见缺陷类型。
- 如何描述 HMI 缺陷。
- 如何定位 HMI 问题。
- 如何把问题转化成经验。

## 常见 HMI 问题类型

### 1. 页面显示问题

常见表现：

- 文案截断
- 图标缺失
- 图片拉伸
- 控件重叠
- 页面错位
- 分辨率适配异常
- 暗色模式下文字看不清

缺陷描述示例：

```text
在夜间模式下，空调页面风量文字颜色过暗，驾驶员无法清晰识别。
```

### 2. 页面跳转问题

常见表现：

- 点击无反应
- 跳错页面
- 返回路径错误
- 页面栈异常
- 连续点击后进入异常页面

示例：

```text
从车辆设置三级页面点击返回，未返回二级页面，而是直接回到首页。
```

### 3. 状态不一致

这是座舱 HMI 的高风险问题。

常见表现：

- UI 显示开启，但车辆实际未开启。
- 车辆信号已变化，UI 未刷新。
- 多屏显示状态不一致。
- 下电上电后状态恢复错误。

示例：

```text
点击开启座椅加热后，中控显示已开启，但车辆信号仍为关闭。
```

### 4. 弹窗和 Toast 问题

常见表现：

- 弹窗不消失
- Toast 堆叠
- 文案错误
- 按钮无效
- 弹窗遮挡重要信息
- 弹窗优先级错误

示例：

```text
导航播报弹窗出现时，低电量告警被遮挡，影响安全提示。
```

### 5. 性能体验问题

常见表现：

- 页面打开慢
- 点击响应慢
- 滑动卡顿
- 动效掉帧
- 页面白屏
- 长时间 loading

示例：

```text
连续切换导航和音乐页面 20 次后，页面切换耗时超过 5 秒。
```

### 6. 多屏联动问题

常见表现：

- 中控状态变了，仪表没变。
- HUD 信息延迟。
- 副驾屏操作影响主驾屏。
- 某一屏重启后状态不同步。

示例：

```text
中控结束导航后，仪表仍显示上一条导航转向信息。
```

### 7. 行驶限制问题

车载场景下，部分功能行驶中应限制。

常见问题：

- 行驶中可播放视频。
- 行驶中可操作复杂设置。
- 行驶中弹出过多干扰信息。
- 危险车控没有二次确认。

示例：

```text
车辆行驶中仍可进入视频全屏播放页面，不符合驾驶安全要求。
```

## HMI 缺陷怎么写

一个好的 HMI 缺陷应该包含：

- 测试环境
- 软件版本
- 前置条件
- 复现步骤
- 实际结果
- 预期结果
- 图片或视频
- 日志或信号证据
- 影响范围

模板：

```markdown
## 缺陷标题

夜间模式下空调页面温度文字显示不清

## 测试环境

- 车型：
- 软件版本：
- 屏幕：
- 模式：夜间模式

## 前置条件

- 车机启动完成。
- 已切换到夜间模式。

## 复现步骤

1. 进入首页。
2. 点击空调卡片。
3. 查看温度显示区域。

## 实际结果

温度文字颜色过暗，与背景对比度不足。

## 预期结果

温度文字应清晰可读，满足夜间模式可视要求。

## 附件

- 截图
- 视频
- 日志
```

## HMI 问题定位思路

### 1. 是设计问题还是实现问题

先对比设计稿：

- 设计稿也是这样 → 可能是设计问题。
- 设计稿正常，实车异常 → 可能是实现问题。

### 2. 是单页面问题还是全局问题

例如字体颜色问题：

- 只有空调页有 → 页面问题。
- 所有夜间模式页面都有 → 主题配置问题。

### 3. 是 UI 问题还是数据问题

例如状态不一致：

- 车辆信号没变 → 车控执行问题。
- 车辆信号变了，UI 没变 → UI 刷新问题。
- 中控变了，仪表没变 → 多屏同步问题。

### 4. 是偶现还是必现

偶现问题要记录：

- 复现概率
- 操作频率
- 持续时间
- 是否和网络、下电、快速点击有关

## 经验沉淀模板

```markdown
## 问题类型

状态不一致

## 问题描述

中控显示座椅加热开启，但车辆实际信号未开启。

## 触发场景

连续快速点击座椅加热开关。

## 根因分析

UI 状态先行更新，但底层车控指令失败后没有回滚。

## 测试经验

车控类 HMI 不能只看 UI，要同时看车辆信号。

## 后续用例补充

- 连续点击车控开关。
- 车控执行失败后 UI 是否回滚。
- 下电上电后状态是否一致。
```

## 面试表达示例

如果面试官问：

```text
你在座舱 HMI 测试中关注哪些点？
```

可以这样回答：

```text
我会从页面显示、页面跳转、控件操作、状态反馈、多屏联动、夜间模式、异常场景和性能体验几个方面测试。车载 HMI 和普通 App 不一样，我会特别关注驾驶安全和车辆真实状态一致性，比如车控类功能不能只看 UI 显示，还要结合车辆信号确认执行结果。
```

## 本节重点

- HMI 缺陷不只是 UI 不好看，也可能影响驾驶安全。
- 状态一致性、多屏联动、行驶限制是车载 HMI 重点。
- 缺陷要带截图、视频、日志或信号证据。
- 每个典型问题都应该沉淀成后续测试用例。
