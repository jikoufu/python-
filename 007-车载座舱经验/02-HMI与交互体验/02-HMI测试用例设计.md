# 02 HMI 测试用例设计

## 学习目标

这一节学习如何设计车载 HMI 测试用例。

你需要掌握：

- HMI 测试从哪些维度设计。
- 页面、控件、弹窗、状态、异常怎么测。
- 如何写一条 HMI 测试用例。
- 如何沉淀 HMI 测试清单。

## HMI 测试设计维度

设计 HMI 用例时，可以按这个顺序：

```text
页面显示
    ↓
页面跳转
    ↓
控件操作
    ↓
状态反馈
    ↓
数据刷新
    ↓
弹窗 Toast
    ↓
多屏联动
    ↓
主题模式
    ↓
异常场景
    ↓
性能体验
```

## 1. 页面显示测试

测试点：

- 页面是否正常加载。
- 字体、图标、图片是否显示正常。
- 页面是否有错位、重叠、截断。
- 重要信息是否完整。
- 横竖屏或不同分辨率是否适配。

示例用例：

```text
打开空调页面，检查温度、风量、模式、开关状态是否完整显示。
```

## 2. 页面跳转测试

测试点：

- 点击入口是否跳转到正确页面。
- 返回按钮是否回到上一级。
- Home 键是否回到首页。
- 多层菜单返回路径是否正确。
- 快捷入口是否有效。

示例：

```text
首页点击“车辆设置” → 进入车辆设置页 → 点击返回 → 回到首页。
```

## 3. 控件操作测试

常见控件：

- 按钮
- 开关
- 滑块
- 列表
- Tab
- 输入框
- 下拉选择
- 卡片

测试点：

- 点击是否生效。
- 状态是否变化。
- 连续点击是否异常。
- 禁用状态是否不可点击。
- 控件边界是否能正确操作。

示例：

```text
拖动音量滑块到最大，检查音量 UI 显示最大，实际音量变化。
```

## 4. 状态反馈测试

用户操作后必须有反馈。

测试点：

- 开关状态是否更新。
- Loading 是否出现和消失。
- 成功提示是否正确。
- 失败提示是否清晰。
- 车辆真实状态和 UI 状态是否一致。

示例：

```text
点击打开座椅加热，UI 显示开启，座椅加热状态信号同步为开启。
```

## 5. 数据刷新测试

座舱页面很多数据来自车辆信号或云端。

测试点：

- 数据是否及时刷新。
- 网络异常时是否保留旧数据或提示失败。
- 车辆信号变化后 UI 是否同步。
- 刷新失败是否有降级展示。

示例：

```text
车辆速度变化时，仪表速度显示同步更新。
```

## 6. 弹窗和 Toast 测试

测试点：

- 弹窗触发条件是否正确。
- 弹窗内容是否准确。
- 确认和取消按钮是否生效。
- Toast 是否自动消失。
- 多个弹窗同时触发时优先级是否正确。
- 弹窗是否遮挡驾驶关键信息。

示例：

```text
行驶中点击禁止操作功能，系统弹出“行驶中不可操作”提示。
```

## 7. 多屏联动测试

测试点：

- 中控操作后仪表是否同步。
- 中控导航后 HUD 是否同步。
- 副驾屏操作是否影响主驾安全信息。
- 某一屏重启后其他屏是否正常。

示例：

```text
中控开始导航，仪表显示下一转向信息，HUD 显示距离和方向。
```

## 8. 主题和模式测试

测试点：

- 日间/夜间模式切换。
- 自动模式根据时间或灯光切换。
- 主题切换后图标和字体是否清楚。
- 多屏主题是否一致。
- 地图主题是否联动。

示例：

```text
切换到夜间模式，检查中控、仪表、HUD 主题同步变暗。
```

## 9. 异常场景测试

常见异常：

```text
弱网
断网
接口超时
车辆信号丢失
连续点击
页面快速切换
系统重启
应用崩溃恢复
下电再上电
```

示例：

```text
断网后进入在线音乐页面，页面应提示网络异常，不应白屏或卡死。
```

## 10. 性能体验测试

测试点：

- 页面打开时间。
- 页面切换是否流畅。
- 滑动是否卡顿。
- 点击响应时间。
- 开机后首页加载时间。
- 动效是否掉帧。

示例：

```text
点击导航入口后，导航页面应在 2 秒内完成首屏显示。
```

## HMI 测试用例模板

```markdown
## 用例编号

HMI_001

## 用例标题

首页点击空调卡片进入空调页面

## 前置条件

- 车机启动完成。
- 当前位于首页。

## 测试步骤

1. 查看首页空调卡片是否显示。
2. 点击空调卡片。
3. 观察页面跳转结果。

## 预期结果

- 空调卡片显示正常。
- 点击后进入空调页面。
- 空调页面温度、风量、模式显示完整。
- 页面无卡顿、无白屏。

## 优先级

高
```

## HMI 测试清单

| 维度 | 检查点 |
|------|--------|
| 页面 | 显示完整、无错位、无截断 |
| 跳转 | 入口正确、返回正确、层级清楚 |
| 控件 | 点击生效、禁用不可点、状态变化 |
| 反馈 | Loading、成功提示、失败提示 |
| 弹窗 | 触发正确、优先级正确、不遮挡 |
| Toast | 文案正确、自动消失、不堆叠 |
| 多屏 | 信息同步、延迟可接受 |
| 主题 | 日夜模式一致、对比度合适 |
| 异常 | 弱网、超时、重启、下电上电 |
| 性能 | 响应快、滑动流畅、无卡死 |

## 本节重点

- HMI 测试要覆盖显示、跳转、控件、状态、异常。
- 车载 HMI 必须关注驾驶安全和多屏联动。
- 状态反馈和车辆真实状态一致非常重要。
- 弹窗、Toast、Loading 是高频问题区域。
