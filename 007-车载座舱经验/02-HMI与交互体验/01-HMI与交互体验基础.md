# 01 HMI 与交互体验基础

## 学习目标

学完这一节，你需要知道：

- HMI 是什么。
- 车载座舱 HMI 包含哪些内容。
- 车载 HMI 和手机 App 的区别。
- 交互体验测试主要看哪些方面。

## HMI 是什么

HMI 是 Human Machine Interface，中文叫人机交互界面。

在车载座舱里，HMI 指用户和车辆系统交互的界面和方式。

它包括：

- 中控屏页面
- 仪表屏显示
- HUD 显示
- 弹窗
- Toast
- 按钮
- 列表
- 卡片
- 手势
- 语音交互入口
- 多屏联动
- 动效和状态反馈

简单说，用户在车里“看到的、点到的、听到的、反馈到的”，很多都属于 HMI 范围。

## 车载 HMI 的特点

车载 HMI 和普通手机 App 最大区别是：

```text
车载 HMI 使用场景在车内，很多操作发生在驾驶过程中。
```

所以它更强调：

- 安全
- 快速
- 清晰
- 少打扰
- 少层级
- 可盲操作
- 状态明确
- 信息优先级

普通 App 可以让用户慢慢看，但车载 HMI 不行。

驾驶员看屏幕的时间越长，风险越高。

## 车载 HMI 常见页面类型

### 1. 首页

首页通常承载高频入口：

- 导航
- 音乐
- 空调
- 电话
- 车辆状态
- 快捷卡片

测试重点：

- 启动后首页是否正常显示。
- 卡片内容是否刷新。
- 快捷入口是否可点击。
- 页面是否有遮挡或错位。

### 2. 设置页

设置页包含大量车辆和系统配置。

测试重点：

- 开关状态是否保存。
- 修改后是否立即生效。
- 多层菜单返回是否正确。
- 车辆下电上电后配置是否保持。

### 3. 车控页

车控页用于控制车辆功能。

例如：

- 空调
- 车窗
- 座椅
- 灯光
- 后视镜

测试重点：

- UI 状态和车辆真实状态是否一致。
- 行驶中是否限制危险操作。
- 操作失败是否有提示。
- 连续点击是否异常。

### 4. 导航页

测试重点：

- 地图加载
- 定位
- 搜索
- 路线规划
- 导航播报
- 仪表/HUD 联动

### 5. 多媒体页

测试重点：

- 播放/暂停
- 上一首/下一首
- 音源切换
- 歌词/封面
- 蓝牙音乐
- 音频焦点

## 交互体验测试关注点

### 1. 页面结构

要看：

- 页面布局是否符合设计。
- 重要信息是否突出。
- 按钮是否容易找到。
- 文案是否清楚。
- 字体是否可读。

### 2. 操作路径

要看：

- 高频功能入口是否足够浅。
- 是否需要过多点击。
- 返回路径是否清晰。
- 页面跳转是否符合预期。

例如调节空调温度，如果要点很多层菜单，体验就不好。

### 3. 状态反馈

用户操作后必须有反馈。

例如：

- 点击按钮后状态变化。
- 加载时显示 loading。
- 操作失败时有错误提示。
- 开关打开后 UI 状态变为开启。

没有反馈会让用户不确定操作是否成功。

### 4. 弹窗和 Toast

弹窗适合重要信息。

Toast 适合轻提示。

测试重点：

- 是否遮挡驾驶关键信息。
- 是否能关闭。
- 是否自动消失。
- 多个弹窗是否冲突。
- 文案是否清晰。

### 5. 夜间模式

夜间模式要关注：

- 亮度是否刺眼。
- 对比度是否足够。
- 图标是否清楚。
- 地图是否切换夜间主题。
- 多屏主题是否一致。

### 6. 多屏联动

智能座舱常见多屏：

- 中控屏
- 仪表屏
- HUD
- 副驾屏
- 后排屏

联动示例：

```text
中控开始导航 → 仪表显示转向信息 → HUD 显示距离和方向
```

测试重点：

- 信息是否同步。
- 延迟是否可接受。
- 多屏显示是否一致。
- 某一屏异常时是否影响其他屏。

### 7. 触控响应

测试重点：

- 点击是否灵敏。
- 滑动是否流畅。
- 长按是否识别。
- 连续点击是否异常。
- 误触是否容易发生。

## 本节重点

- HMI 是人和车机交互的界面和方式。
- 车载 HMI 比普通 App 更关注驾驶安全。
- HMI 测试不仅看功能，也看体验和状态反馈。
- 页面层级、弹窗、夜间模式、多屏联动、触控响应都是重点。
