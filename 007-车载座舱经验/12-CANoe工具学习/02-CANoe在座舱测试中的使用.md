# 02 CANoe 在座舱测试中的使用

## 学习目标

这一节学习 CANoe 在车载座舱测试中的具体用法。

你需要掌握：

- 如何用 CANoe 辅助座舱功能测试。
- 如何验证车辆信号和 UI 状态一致。
- 如何设计 CANoe 相关测试场景。
- 常见座舱模块如何结合 CANoe 测试。

## 座舱测试里 CANoe 的典型用途

### 1. 模拟车辆状态

例如模拟：

```text
车速 = 0
车速 = 30
档位 = P
档位 = D
左前门 = 打开
安全带 = 未系
电量 = 20%
```

然后观察车机：

- 是否显示正确状态。
- 是否触发对应限制。
- 是否出现告警。

### 2. 验证车机操作是否发信号

例如用户在中控屏点击：

```text
打开空调
调节温度
开启座椅加热
切换驾驶模式
打开氛围灯
```

CANoe 可以观察是否有对应控制报文发出。

测试重点：

- 是否发了正确报文。
- 信号值是否正确。
- 发送周期是否符合要求。
- 操作失败时是否有回滚或提示。

### 3. 验证 UI 和信号一致

这是座舱测试高频问题。

示例：

```text
CANoe 中 DoorStatus = Open
车机 UI 应显示车门打开
```

或者：

```text
车机点击打开座椅加热
CANoe 中 SeatHeatRequest 应变为 On
车辆反馈 SeatHeatStatus = On
UI 显示开启
```

## 常见测试场景

### 场景 1：车速限制

测试目标：

验证行驶中是否限制危险操作。

步骤：

```text
1. CANoe 模拟 VehicleSpeed = 0。
2. 在车机打开视频页面。
3. CANoe 模拟 VehicleSpeed = 30。
4. 观察视频是否被限制播放。
```

预期：

```text
车速为 0 时允许操作。
车速大于限制阈值时，视频播放被限制或提示行驶中不可观看。
```

### 场景 2：车门状态显示

测试目标：

验证车门状态 UI 是否正确。

步骤：

```text
1. CANoe 模拟左前门打开。
2. 查看中控车辆状态页面。
3. 查看仪表是否显示左前门打开。
4. CANoe 模拟左前门关闭。
5. 查看 UI 是否恢复关闭状态。
```

预期：

```text
中控和仪表显示一致。
状态变化及时。
无延迟过大、无显示错误。
```

### 场景 3：空调控制

测试目标：

验证中控空调操作和车辆信号一致。

步骤：

```text
1. 打开空调页面。
2. 点击开启空调。
3. CANoe 观察 A/C 请求信号。
4. 模拟空调控制器反馈开启状态。
5. 查看 UI 是否显示开启。
```

预期：

```text
车机发出正确控制请求。
收到反馈后 UI 状态更新。
如果没有反馈，应提示失败或保持原状态。
```

### 场景 4：安全带告警

测试目标：

验证安全带状态和告警显示。

步骤：

```text
1. CANoe 模拟驾驶员安全带未系。
2. CANoe 模拟车速大于阈值。
3. 查看仪表和中控是否有告警。
4. CANoe 模拟安全带已系。
5. 查看告警是否消失。
```

预期：

```text
满足条件时触发告警。
状态恢复后告警消失。
```

### 场景 5：下电上电状态恢复

测试目标：

验证车辆状态变化后重启是否正常恢复。

步骤：

```text
1. 设置某个车控状态。
2. 记录 CANoe 中信号值。
3. 模拟车辆下电。
4. 模拟车辆上电。
5. 查看 UI 状态和信号状态是否一致。
```

预期：

```text
应按需求恢复默认状态或上次记忆状态。
UI 和信号一致。
```

## CANoe 测试用例模板

```markdown
## 用例编号

CANOE_HMI_001

## 用例标题

车速大于 30 km/h 时限制视频播放

## 前置条件

- CANoe 工程已加载。
- DBC 已正确配置。
- 车机和 CANoe 通信正常。
- 当前车辆处于上电状态。

## 测试步骤

1. 使用 CANoe 设置 VehicleSpeed = 0。
2. 进入车机视频页面。
3. 确认视频可播放。
4. 使用 CANoe 设置 VehicleSpeed = 30。
5. 观察车机视频播放状态。

## 预期结果

- 车速为 0 时视频可播放。
- 车速为 30 时视频播放被限制。
- 页面提示“行驶中不可观看”或符合需求文案。

## 检查点

- CANoe 中 VehicleSpeed 信号值正确。
- 车机 UI 状态正确。
- 限制逻辑符合需求。
```

## CANoe 问题定位思路

### 1. UI 没变，先看信号有没有变

如果 CANoe 中信号没变：

```text
可能是信号没发、仿真配置错、DBC 错、总线通信异常。
```

如果 CANoe 中信号变了，但 UI 没变：

```text
可能是车机没接收、信号映射错误、UI 刷新逻辑异常。
```

### 2. UI 变了，实际信号没变

可能是：

```text
UI 先行更新
控制报文未发出
底层执行失败
反馈信号未校验
```

这类问题在车控测试中很重要。

### 3. 多屏显示不一致

需要分别看：

```text
中控接收的信号
仪表接收的信号
HUD 接收的信号
各屏 UI 刷新时机
```

## 学习 CANoe 的建议路线

1. 先理解 CAN、ECU、报文、信号、DBC。
2. 学会打开 CANoe 工程。
3. 学会看 Trace Window。
4. 学会看 Data Window。
5. 学会修改和发送信号。
6. 学会使用 Panel 模拟场景。
7. 学会保存日志和测量文件。
8. 再学习 CAPL 和自动化测试。

## 本节重点

- CANoe 在座舱测试中常用来模拟车辆信号。
- 车控测试不能只看 UI，要看请求信号和反馈信号。
- 车速、档位、车门、安全带、空调、座椅是常见测试信号。
- 问题定位要区分信号没变、车机没接收、UI 没刷新、执行没反馈。
