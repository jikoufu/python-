# 01 车载总线学习计划（三周）

## 学习目标定位

这个岗位总线是**辅助技能**，不是主战场。你要达到的程度是：

- 面试能聊明白原理
- 实际工作中能看懂报文
- 能和总线工程师对话
- 出问题时能配合定位

**不需要**会做总线一致性测试、网络管理开发这些深度内容。

## 总投入

```text
总时长：约 3 周
每天：1 ~ 1.5 小时
```

## 第 1 周：CAN / CAN FD（重点中的重点）

### Day 1-2：CAN 基础概念

- [ ] CAN 是什么、为什么用它
- [ ] 总线拓扑：双绞线、120Ω 终端电阻
- [ ] 多主、广播、基于消息（不是基于地址）的通信特点
- [ ] 显性位 / 隐性位、差分信号

> 配套：`02-CAN与CANFD详解.md` 第一、二部分

### Day 3-4：CAN 帧结构（必须吃透）

- [ ] 数据帧组成：帧起始、仲裁场(ID)、控制场、数据场、CRC、ACK、帧结束
- [ ] 标准帧（11 位 ID） vs 扩展帧（29 位 ID）
- [ ] 仲裁机制：ID 越小优先级越高，怎么"非破坏性仲裁"

> 配套：`02-CAN与CANFD详解.md` 第三部分

### Day 5：CAN FD 与错误处理

- [ ] CAN FD 相比 CAN 的改进：数据场最多 64 字节、可变速率
- [ ] 错误类型、错误帧、错误计数器、总线关闭（了解即可）

> 配套：`02-CAN与CANFD详解.md` 第四、五部分

### Day 6-7：动手——读报文（最重要）

- [ ] 理解 DBC 文件：Message（报文）、Signal（信号）
- [ ] 如何从原始字节解析出物理值
- [ ] 信号的起始位、长度、字节序（Intel/Motorola）、缩放和偏移
- [ ] 用 Python 的 `cantools` 库实操解析

> 配套：`05-DBC报文解析实战.md`
> 这步对你最重要，座舱测试中你会大量看信号。

## 第 2 周：LIN + 车载以太网

### Day 8-9：LIN（轻量了解）

- [ ] 主从架构（一个主节点，多个从节点）
- [ ] 应用场景：车窗、座椅、灯光等低速低成本场景
- [ ] 帧结构概念（同步、PID、数据、校验）——知道即可
- [ ] 记住一句话：**LIN 是 CAN 的低成本补充**

> 配套：`03-LIN与车载以太网.md` 第一部分

### Day 10-13：车载以太网（座舱重点）

座舱大量用以太网，这块比 LIN 重要得多。

- [ ] 为什么座舱/智驾用以太网：大带宽传图像、音视频、地图
- [ ] 车载以太网 vs 普通以太网（物理层 100BASE-T1 / 1000BASE-T1，单对双绞线）
- [ ] **SOME/IP**：面向服务的通信，理解"服务发现、方法调用、事件订阅"
- [ ] DoIP：基于以太网的诊断（了解）
- [ ] TSN（时间敏感网络）：知道名词和用途即可

> 配套：`03-LIN与车载以太网.md` 第二、三部分

### Day 14：网关与整车通信

- [ ] 网关如何在 CAN/LIN/以太网之间转换路由
- [ ] 数据在不同域之间怎么流转

> 配套：`03-LIN与车载以太网.md` 第四部分

## 第 3 周：AUTOSAR 基础 + 工具入门

### Day 15-17：AUTOSAR（理解架构，不深入）

- [ ] 为什么需要 AUTOSAR：软硬件解耦、标准化、复用
- [ ] **CP（Classic Platform）**：传统 ECU 分层架构（应用层、RTE、BSW、MCAL），理解分层思想
- [ ] **AP（Adaptive Platform）**：面向高性能计算（智驾、座舱），基于 POSIX、面向服务
- [ ] 核心记忆：**CP 管实时控制类 ECU，AP 管算力大的智能化 ECU**

> 配套：`04-AUTOSAR基础.md`

### Day 18-20：工具入门（CANoe）

- [ ] CANoe 是什么、行业地位
- [ ] 跟视频做：加载 DBC、Trace 窗口看报文、Bus Statistics 看总线负载
- [ ] 了解怎么发送报文、怎么仿真节点（看懂即可，不要求精通）

> 配套：`../12-CANoe工具学习/`（本仓库已有 CANoe 专门模块）

### Day 21：串联总结 + 面试准备

- [ ] 把整条链路讲一遍：传感器 → 域内 CAN → 域控制器 → 以太网骨干 → 网关 → 其他域
- [ ] 准备面试话术：CAN 帧结构、CAN vs 以太网应用区别、AUTOSAR 分层

> 配套：`06-面试题汇总.md`

## 几条重要提醒

**别在 LIN 和 AUTOSAR 上花太多时间。** 它们是"了解"级别，CAN 和车载以太网才值得多投入——尤其车载以太网，座舱是以太网重灾区。

**DBC 读报文是性价比最高的技能。** 零成本、面试能展示、实际工作天天用。用 Python 的 `cantools` 库练习解析，结合你的 Python 优势，这是能写进简历的小亮点。

**没有硬件也能学。** CANoe 有 Demo 版和仿真模式，cantools 纯软件就能跑，不用买硬件。
