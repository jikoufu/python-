# 01 CANoe 是什么

## 学习目标

学完这一节，你需要知道：

- CANoe 是什么。
- CANoe 在车载测试中做什么。
- CANoe 和 CAN 总线、ECU、DBC 的关系。
- 座舱测试为什么经常要求会 CANoe。

## CANoe 是什么

CANoe 是 Vector 公司的一款车载网络开发和测试工具。

它常用于：

- 总线报文分析
- ECU 仿真
- 节点仿真
- 信号收发
- 诊断测试
- 自动化测试
- 车载网络问题定位

简单理解：

```text
CANoe 可以让测试人员看到车上总线在发什么，也可以模拟一些车辆信号发给被测系统。
```

## 为什么车载测试要用 CANoe

车载座舱不是一个独立 App。

它需要和车辆信号交互：

```text
车速
档位
车门状态
安全带状态
空调状态
座椅状态
灯光状态
电量 / 油量
故障码
```

这些信号很多来自车载网络。

如果没有工具，很难验证：

- 车机显示的状态是否正确。
- UI 操作是否真的发出车控信号。
- 车辆信号变化后车机是否同步。
- 某个功能是否在特定车速或档位下被限制。

CANoe 就是用来观察和模拟这些信号的重要工具。

## 必懂基础概念

### 1. CAN

CAN 是 Controller Area Network，是车载网络中常见的通信总线。

很多 ECU 会通过 CAN 交换数据。

例如：

```text
BCM 发送车门状态
空调控制器发送空调状态
仪表接收车速和告警
座舱域控接收车辆状态
```

### 2. ECU

ECU 是 Electronic Control Unit，电子控制单元。

可以理解为车上的控制模块。

例如：

- BCM 车身控制模块
- HVAC 空调控制器
- TBOX 车联网模块
- IVI 车机
- Cluster 仪表
- ADAS 控制器

### 3. 报文

CAN 总线上传输的是报文。

报文里包含多个信号。

例如一个车门状态报文可能包含：

```text
左前门状态
右前门状态
左后门状态
右后门状态
后备箱状态
```

### 4. 信号

信号是报文中的具体字段。

例如：

```text
VehicleSpeed = 60
Gear = D
DriverDoorStatus = Open
SeatBeltStatus = Buckled
```

测试时经常说“发一个车速信号”，本质上就是通过 CANoe 模拟对应报文中的信号值。

### 5. DBC

DBC 是 CAN 数据库文件。

它描述：

- 报文 ID
- 报文名称
- 信号名称
- 信号起始位
- 信号长度
- 信号缩放系数
- 枚举含义

有了 DBC，CANoe 才能把原始报文解析成人能看懂的信号。

例如：

```text
0x123 原始数据：08 00 00 00 ...
解析后：VehicleSpeed = 8 km/h
```

## CANoe 常见窗口

不同项目配置会不一样，但常见有：

### Trace Window

查看总线报文。

用于分析：

- 报文是否发送。
- 报文周期是否正常。
- 信号值是否变化。
- 是否有异常报文。

### Graphics Window

以曲线方式查看信号变化。

适合观察：

- 车速变化
- 电压变化
- 温度变化
- 某个状态随时间变化

### Data Window

查看信号当前值。

适合测试时实时观察：

```text
VehicleSpeed 当前是多少？
Gear 当前是什么？
DoorStatus 当前是什么？
```

### Measurement Setup

配置测量环境。

包括：

- 总线通道
- 数据库 DBC
- 节点
- 仿真模块
- 测试模块

### Panel

面板可以做成可视化操作界面。

例如：

- 输入车速
- 切换档位
- 打开车门
- 发送故障状态

测试人员可以通过 Panel 快速模拟车辆状态。

## CANoe 在座舱测试中的价值

座舱测试中，CANoe 常用于验证：

```text
车辆信号 → 车机显示
车机操作 → 车辆信号
车辆状态 → 功能限制
故障信号 → 告警显示
诊断请求 → ECU 响应
```

例如：

```text
CANoe 模拟车速 = 30 km/h
    ↓
车机应限制视频播放
```

或者：

```text
CANoe 模拟左前门打开
    ↓
中控和仪表应显示左前门打开
```

## 本节重点

- CANoe 是车载网络测试工具。
- 它能看报文，也能模拟信号。
- DBC 用来把原始报文解析成信号。
- 座舱测试常用 CANoe 验证 UI 与车辆信号是否一致。
- 面试说会 CANoe，至少要能讲清 Trace、Data、Panel、DBC、信号仿真这些概念。
