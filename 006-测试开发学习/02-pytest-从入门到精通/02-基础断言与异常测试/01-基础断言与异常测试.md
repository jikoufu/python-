# 02 基础断言与异常测试

## 是什么

断言就是判断程序结果是否符合预期。pytest 直接使用 Python 原生 `assert`，不需要记很多专门的断言方法。

```python
def test_equal():
    assert 1 + 1 == 2
```

异常测试用于验证代码在错误输入下是否会按预期抛出异常。

```python
import pytest


def divide(a, b):
    if b == 0:
        raise ValueError("b cannot be zero")
    return a / b


def test_divide_zero():
    with pytest.raises(ValueError):
        divide(1, 0)
```

## 为什么要用它

测试不只是验证正常路径，也要验证异常路径。真实业务中很多严重问题都来自边界条件：

- 空字符串
- None
- 负数
- 超长输入
- 非法状态
- 第三方返回异常

断言越清晰，失败时越容易定位问题。

## 怎么用

常见断言：

```python
assert actual == expected
assert name in users
assert result is True
assert data is not None
assert isinstance(data, dict)
assert len(items) == 3
```

浮点数近似比较：

```python
import pytest


def test_float():
    assert 0.1 + 0.2 == pytest.approx(0.3)
```

异常信息断言：

```python
import pytest


def test_error_message():
    with pytest.raises(ValueError, match="cannot be zero"):
        divide(1, 0)
```

## 场景例子

场景：用户注册时校验手机号。

```python
def validate_phone(phone):
    if not phone:
        raise ValueError("phone is required")
    if len(phone) != 11:
        raise ValueError("phone length invalid")
    return True


def test_validate_phone_success():
    assert validate_phone("13800138000") is True


def test_validate_phone_empty():
    with pytest.raises(ValueError, match="required"):
        validate_phone("")
```

## 大厂使用的例子

大型业务系统通常会把边界断言作为质量底线。例如用户、订单、支付、权限这类模块，测试不仅覆盖成功路径，还会专门覆盖：

- 参数缺失
- 参数类型错误
- 金额精度问题
- 权限不足
- 状态流转非法

这样可以在代码合并前提前发现“线上才会暴露”的边界缺陷。

## 本章小结

基础断言是 pytest 的日常基本功。写测试时不要只测“正常情况”，要刻意补充异常输入、边界输入和错误状态，这才是测试真正产生价值的地方。
