# 01 环境准备与测试认知

## 是什么

pytest 是 Python 生态中最常用的测试框架之一。它可以用很少的代码编写测试用例，并自动发现、执行、汇总测试结果。

最小测试示例：

```python
def add(a, b):
    return a + b


def test_add():
    assert add(1, 2) == 3
```

## 为什么要用它

相比直接写脚本检查结果，pytest 的优势是：

- 自动发现测试文件和测试函数。
- 失败时给出清晰的断言信息。
- 支持 fixture、参数化、mark、插件、报告和 CI。
- 适合从小函数单元测试扩展到大型自动化测试框架。

对测试开发来说，pytest 是自动化测试的底座。后面的接口测试、AI 接口评测、CI 回归，本质上都可以建立在 pytest 上。

## 怎么用

安装：

```bash
pip install pytest
```

推荐先创建虚拟环境：

```bash
python -m venv .venv
.\.venv\Scripts\activate
pip install pytest
```

运行测试：

```bash
pytest
pytest -q
pytest -v
pytest tests/test_demo.py
pytest tests/test_demo.py::test_add
```

pytest 默认识别：

- `test_*.py`
- `*_test.py`
- `test_` 开头的函数
- `Test` 开头的测试类

## 场景例子

场景：你写了一个价格计算函数，需要保证后续改代码不会把金额算错。

```python
def calc_price(price, count):
    return price * count


def test_calc_price():
    assert calc_price(10, 3) == 30
```

这类测试适合放在每次提交代码前运行，快速发现基础逻辑问题。

## 大厂使用的例子

大型团队通常会把 pytest 放进代码提交链路：

- 开发者本地提交前运行单元测试。
- Pull Request 自动触发 pytest。
- 核心模块要求测试通过才能合并。
- 测试结果和覆盖率进入质量看板。

比如一个后端团队修改订单金额计算逻辑时，CI 会自动跑价格、优惠券、库存、支付相关测试，避免一个局部改动影响核心交易链路。

## 本章小结

pytest 的第一步不是复杂框架，而是能稳定创建测试文件、运行测试、看懂失败结果。先把“能跑起来”练熟，后面的 fixture、参数化、报告和 CI 才有意义。
