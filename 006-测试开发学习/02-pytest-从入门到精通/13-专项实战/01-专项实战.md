# 13 专项实战

## 是什么

专项实战是把 pytest 用到真实项目中。只学语法不够，必须用项目把测试策略、用例设计、数据管理、报告和 CI 串起来。

推荐实战方向：

- Python 工具函数单元测试。
- FastAPI 接口测试。
- 通用接口自动化测试框架。
- AI 聊天接口测试。
- RAG 问答评测。
- Agent 工具调用测试。

## 为什么要用它

面试和工作中，别人更关心你能不能解决真实问题：

- 怎么设计测试目录？
- 怎么管理测试数据？
- 怎么处理登录鉴权？
- 怎么生成报告？
- 怎么接入 CI？
- 怎么定位失败？
- 怎么保证测试稳定？

实战项目能把这些能力串起来。

## 怎么用

实战 1：FastAPI 接口测试。

```text
目标：
- 用 FastAPI 写几个接口。
- 用 TestClient 编写 pytest 测试。
- 覆盖成功、失败、参数错误。
- 生成测试报告。
```

实战 2：接口自动化框架。

```text
目标：
- 封装 HttpClient。
- fixture 管理登录 token。
- 参数化读取测试数据。
- 输出 HTML 和 JUnit XML 报告。
- 接入 GitHub Actions。
```

实战 3：AI 聊天接口测试。

```text
目标：
- 测试 /chat 接口。
- 断言响应结构。
- 断言 answer 不为空。
- 断言敏感问题拒答。
- 统计响应时间。
```

## 场景例子

AI 聊天接口测试：

```python
def test_chat_answer_not_empty(api_client):
    response = api_client.post(
        "/chat",
        json={"message": "pytest 是什么？"},
    )
    assert response.status_code == 200
    data = response.json()
    assert data["answer"]
    assert "pytest" in data["answer"].lower()
```

RAG 问答评测：

```python
def test_rag_answer_has_source(api_client):
    response = api_client.post(
        "/rag/query",
        json={"question": "公司的报销流程是什么？"},
    )
    data = response.json()
    assert data["answer"]
    assert data["sources"]
```

## 大厂使用的例子

大型团队会把实战项目做成长期资产：

- 接口自动化项目守护核心服务。
- AI 评测项目守护 Prompt 和知识库质量。
- 回归测试项目守护发布流程。
- 稳定性测试项目发现 flaky 和慢用例。

例如 AI 应用团队会在每次 Prompt 改动后自动运行一批问题集，比较回答格式、拒答策略、召回来源和耗时成本。

实战项目不要贪大。一个小而完整的项目，比只有很多零散笔记更有价值。建议至少完成“接口自动化框架”和“AI 接口测试框架”两个作品。
