# 09 并发、重跑与稳定性治理

## 是什么

并发运行用于加速测试，失败重跑用于缓解偶发失败，稳定性治理用于消除 flaky 用例。

常用插件：

- `pytest-xdist`：并发执行。
- `pytest-rerunfailures`：失败重跑。

## 为什么要用它

测试规模变大后，会出现两个问题：

- 全量测试太慢。
- 部分用例偶发失败，影响团队信任。

并发可以缩短反馈时间，但稳定性治理比简单重跑更重要。重跑只能缓解问题，不能替代修复问题。

## 怎么用

安装：

```bash
pip install pytest-xdist pytest-rerunfailures
```

并发运行：

```bash
pytest -n auto
pytest -n 4
```

失败重跑：

```bash
pytest --reruns 2 --reruns-delay 1
```

给慢测试打标：

```python
import pytest


@pytest.mark.slow
def test_big_data_import():
    assert True
```

## 场景例子

场景：接口自动化测试有 600 条用例，串行运行 30 分钟，并发后缩短到 8 分钟。

```bash
pytest tests/api -n auto --reruns 1
```

同时需要检查：

- 用例是否共享同一条测试数据。
- 并发执行是否互相污染。
- 测试账号、订单号、文件名是否唯一。

## 大厂使用的例子

大型团队常见做法：

- 单元测试默认并发运行。
- 接口测试按模块拆分并发。
- 高风险 flaky 用例进入治理清单。
- 重跑次数有限，避免掩盖真实问题。
- 对测试耗时做排行榜，持续优化慢用例。

例如订单测试中，每个并发用例都创建独立订单号，测试完成后清理，避免多个用例同时改同一条订单。

## 本章小结

并发让测试更快，稳定性治理让测试更可信。不要只靠失败重跑粉饰问题，要从测试隔离、数据唯一性、外部依赖、等待策略上真正解决 flaky。
