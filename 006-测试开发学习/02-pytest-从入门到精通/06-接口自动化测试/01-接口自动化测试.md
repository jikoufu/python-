# 06 接口自动化测试

## 是什么

接口自动化测试是用代码请求 HTTP 接口，并断言响应是否符合预期。pytest 负责组织测试，requests 或 httpx 负责发请求。

```python
import requests


def test_get_user():
    response = requests.get("http://127.0.0.1:8000/users/1")
    assert response.status_code == 200
    assert response.json()["id"] == 1
```

## 为什么要用它

接口是前端、后端、移动端、AI 应用之间的核心契约。接口自动化可以：

- 快速验证接口是否可用。
- 覆盖鉴权、参数、状态码、响应格式。
- 在 CI 中持续回归。
- 比 UI 自动化更稳定、更快。

## 怎么用

安装：

```bash
pip install pytest requests httpx
```

GET 请求：

```python
def test_get_users(base_url):
    response = requests.get(f"{base_url}/users")
    assert response.status_code == 200
    assert isinstance(response.json(), list)
```

POST 请求：

```python
def test_create_user(base_url):
    payload = {"username": "alice", "password": "abc123"}
    response = requests.post(f"{base_url}/users", json=payload)
    assert response.status_code == 201
    assert response.json()["username"] == "alice"
```

鉴权：

```python
def test_profile(base_url, auth_headers):
    response = requests.get(f"{base_url}/profile", headers=auth_headers)
    assert response.status_code == 200
```

FastAPI 本地测试：

```python
from fastapi.testclient import TestClient
from main import app


client = TestClient(app)


def test_read_root():
    response = client.get("/")
    assert response.status_code == 200
```

## 场景例子

场景：测试登录接口。

```python
def test_login_success(base_url):
    response = requests.post(
        f"{base_url}/login",
        json={"username": "admin", "password": "123456"},
    )
    assert response.status_code == 200
    data = response.json()
    assert "token" in data
    assert data["token"]
```

## 大厂使用的例子

大型团队通常把接口自动化作为服务质量门禁：

- PR 合并前跑核心接口测试。
- 发布前跑冒烟接口测试。
- 夜间跑全量回归接口测试。
- 对支付、订单、用户、权限等核心服务做强覆盖。

对于 AI 应用，接口自动化还会验证输出结构、响应耗时、错误码、模型回答是否为空、是否满足格式要求。
