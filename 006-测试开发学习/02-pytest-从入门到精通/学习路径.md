# pytest 从入门到精通学习路径

## 阶段 1：环境准备与测试认知

目标：知道 pytest 是什么，能跑通第一个测试。

学习内容：

- pytest 的定位
- pytest 和 unittest 的区别
- 虚拟环境
- 安装 pytest
- 测试文件和测试函数命名规则
- `assert` 断言
- `pytest -q`、`pytest -v`、指定文件运行

阶段成果：

- 能创建一个最小 pytest 项目。
- 能运行单个测试文件、单个测试类、单个测试函数。
- 能看懂测试通过、失败、跳过的输出。

练习任务：

1. 写一个 `add` 函数并测试正常相加。
2. 写一个除法函数并测试除数为 0 的异常。
3. 分别用普通模式、详细模式、指定函数模式运行测试。

## 阶段 2：基础断言与异常测试

目标：掌握 pytest 最常用的判断方式。

学习内容：

- 相等断言
- 包含断言
- 类型断言
- 布尔断言
- 浮点数近似断言
- `pytest.raises`
- 失败信息分析

阶段成果：

- 能为普通 Python 函数写基础单元测试。
- 能验证异常是否按预期抛出。
- 能根据失败输出快速定位问题。

练习任务：

1. 测试字符串处理函数。
2. 测试列表去重函数。
3. 测试金额计算函数，处理浮点误差。
4. 测试非法参数抛出 `ValueError`。

## 阶段 3：fixture 核心能力

目标：理解 pytest 最重要的资源管理机制。

学习内容：

- fixture 是什么
- fixture 返回测试数据
- fixture 管理前置条件
- `yield` 实现后置清理
- fixture 作用域：function、class、module、session
- fixture 依赖 fixture
- `autouse=True`
- `conftest.py`

阶段成果：

- 能用 fixture 管理登录 token、测试用户、数据库连接、临时文件。
- 能把公共 fixture 放到 `conftest.py`。
- 能区分不同 scope 对测试执行的影响。

练习任务：

1. 用 fixture 提供用户测试数据。
2. 用 fixture 创建临时文件并在测试结束后删除。
3. 用 session 级 fixture 模拟全局登录。
4. 让一个 fixture 依赖另一个 fixture。

## 阶段 4：参数化与数据驱动

目标：用少量代码覆盖多组输入和边界条件。

学习内容：

- `@pytest.mark.parametrize`
- 单参数和多参数参数化
- 参数化 ID
- 参数组合
- fixture 参数化
- 从 JSON/YAML/CSV 读取测试数据
- 参数化的可读性设计

阶段成果：

- 能批量测试正常值、边界值、异常值。
- 能把测试数据从代码中拆出来。
- 能为接口自动化准备数据驱动能力。

练习任务：

1. 参数化测试登录接口的账号、密码组合。
2. 参数化测试手机号、邮箱、身份证等输入校验。
3. 从 JSON 文件读取接口测试用例。
4. 给参数化用例设置清晰的用例名称。

## 阶段 5：mark、跳过、预期失败与用例选择

目标：学会给测试分类，并控制测试执行范围。

学习内容：

- `@pytest.mark.smoke`
- `@pytest.mark.regression`
- `pytest -m`
- `pytest -k`
- `pytest.mark.skip`
- `pytest.mark.skipif`
- `pytest.mark.xfail`
- 自定义 mark 注册

阶段成果：

- 能区分冒烟测试、回归测试、慢速测试。
- 能按标签运行部分测试。
- 能合理处理暂时失败或条件不满足的用例。

练习任务：

1. 给核心接口加 `smoke` 标记。
2. 给慢速用例加 `slow` 标记。
3. 只运行登录相关用例。
4. 给已知缺陷用例加 `xfail`。

## 阶段 6：接口自动化测试

目标：用 pytest 完成真实 HTTP 接口自动化。

学习内容：

- HTTP 状态码
- requests / httpx
- GET / POST / PUT / DELETE
- Header、Cookie、Token
- JSON 请求体
- 响应 JSON 断言
- 响应时间断言
- 接口依赖
- FastAPI `TestClient`

阶段成果：

- 能编写完整接口自动化测试。
- 能处理登录鉴权和接口依赖。
- 能测试本地 FastAPI 应用和远程 HTTP 服务。

练习任务：

1. 测试用户查询接口。
2. 测试登录接口成功和失败场景。
3. 用 fixture 管理 token。
4. 测试一个完整业务流程：登录、创建资源、查询资源、删除资源。

## 阶段 7：Mock 与测试隔离

目标：让测试不依赖不稳定的外部环境。

学习内容：

- 为什么需要 Mock
- `monkeypatch`
- `unittest.mock`
- Mock 第三方接口
- Mock 环境变量
- Mock 时间
- Mock 文件读写
- Mock 数据库访问

阶段成果：

- 能隔离外部 API、数据库、时间和环境变量。
- 能提高单元测试稳定性和速度。
- 能判断什么时候该 Mock，什么时候该做集成测试。

练习任务：

1. Mock 一个支付接口返回成功。
2. Mock 环境变量读取配置。
3. Mock 当前时间测试过期逻辑。
4. Mock 数据库查询结果。

## 阶段 8：测试配置、日志与报告

目标：把测试从零散脚本提升为可交付结果。

学习内容：

- `pytest.ini`
- `pyproject.toml` 中配置 pytest
- `conftest.py` 组织方式
- 日志输出
- `pytest-html`
- Allure 报告
- `pytest-cov` 覆盖率
- JUnit XML 报告
- 失败截图和失败日志

阶段成果：

- 能统一配置测试运行参数。
- 能生成可读测试报告。
- 能统计代码覆盖率。
- 能为 CI/CD 提供标准报告文件。

练习任务：

1. 配置 pytest 默认运行路径和 mark。
2. 生成 HTML 测试报告。
3. 生成覆盖率报告。
4. 把失败日志写入报告。

## 阶段 9：并发、重跑与稳定性治理

目标：让测试套件更快、更稳定。

学习内容：

- `pytest-xdist` 并发执行
- `pytest-rerunfailures` 失败重跑
- flaky 用例原因分析
- 测试隔离
- 测试顺序依赖治理
- 临时目录和临时数据
- 超时控制

阶段成果：

- 能提升大型测试套件执行速度。
- 能发现并治理不稳定测试。
- 能避免用例之间互相污染。

练习任务：

1. 用 xdist 并发运行测试。
2. 给偶发失败用例配置重跑。
3. 找出一个有顺序依赖的用例并改造。
4. 给慢接口测试设置超时策略。

## 阶段 10：CI/CD 集成

目标：让 pytest 自动在流水线中运行。

学习内容：

- Git 基础
- GitHub Actions
- GitLab CI
- Jenkins
- 安装依赖
- 执行测试
- 保存报告
- PR 自动回归
- 定时回归

阶段成果：

- 能在代码提交后自动运行 pytest。
- 能在 CI 中保存测试报告和覆盖率报告。
- 能把测试结果作为上线质量门禁。

练习任务：

1. 写一个 GitHub Actions 工作流运行 pytest。
2. 在流水线中生成 JUnit XML。
3. 上传 HTML 报告为构建产物。
4. 配置失败时阻断合并。

## 阶段 11：pytest Hook 与插件开发

目标：理解 pytest 扩展机制，具备高级定制能力。

学习内容：

- pytest 生命周期
- 常用 Hook
- `pytest_addoption`
- 自定义命令行参数
- `pytest_configure`
- `pytest_collection_modifyitems`
- `pytest_runtest_makereport`
- 简单插件开发

阶段成果：

- 能给 pytest 增加自定义命令行参数。
- 能在收集用例时动态打标或过滤。
- 能在用例失败时自动收集日志、截图、接口响应。

练习任务：

1. 增加 `--env` 参数区分 dev、test、prod 环境。
2. 根据命令行参数选择 base_url。
3. 用 Hook 在失败时记录失败原因。
4. 写一个最小 pytest 插件。

## 阶段 12：自动化测试框架封装

目标：形成可维护、可复用、可扩展的测试工程结构。

学习内容：

- 分层设计
- 配置管理
- 日志管理
- HTTP Client 封装
- 公共断言
- 测试数据管理
- 用例组织
- 报告归档
- 环境切换
- 敏感信息管理

阶段成果：

- 能搭建一个公司项目可用的 pytest 自动化测试框架。
- 能让新人按规范快速新增用例。
- 能支撑多环境、多模块、多测试类型。

推荐结构：

```text
api_test_framework/
├── pytest.ini
├── requirements.txt
├── tests/
│   ├── conftest.py
│   ├── test_auth.py
│   ├── test_user.py
│   └── test_order.py
├── common/
│   ├── http_client.py
│   ├── logger.py
│   ├── assertions.py
│   └── data_loader.py
├── config/
│   ├── dev.yaml
│   └── test.yaml
├── data/
│   └── api_cases.yaml
└── reports/
```

练习任务：

1. 封装统一 HTTP 请求类。
2. 封装 JSON 字段断言。
3. 支持不同环境配置切换。
4. 支持测试报告自动归档。

- 数据库读写测试
- AI 聊天接口测试
- RAG 检索与回答质量测试
- Agent 工具调用测试

阶段成果：

- 至少完成 2 个可展示项目。
- 能讲清楚测试策略、框架结构、关键用例、报告和 CI 流程。

推荐作品：

2. FastAPI 项目 pytest 测试套件。

4. RAG 问答评测 pytest 项目。

目标：从会用提升到能讲清楚原理和取舍。

学习内容：

- pytest 测试发现机制

- fixture scope 生命周期
- 参数化生成用例机制
- mark 和 collection
- Hook 插件机制
- 常见面试题
- 框架设计题

阶段成果：

- 能回答 pytest 高频面试题。
- 能解释 fixture、conftest.py、参数化、Hook 的底层逻辑。
- 能设计一个适合业务项目的自动化测试框架。

面试重点：

- pytest 和 unittest 的区别。
- fixture 和 setup/teardown 的区别。
- `conftest.py` 的作用。
- fixture 作用域有哪些。
- 参数化怎么做。
- 接口自动化框架怎么设计。
- 怎么处理接口依赖。
- 怎么处理测试数据。
- 怎么治理 flaky 用例。
- pytest 怎么接入 CI/CD。

## 总结路径

```text
入门
  ↓
基础断言和运行方式
  ↓
fixture 和参数化
  ↓
接口自动化
  ↓
Mock 与隔离
  ↓
报告、覆盖率、插件
  ↓
CI/CD
  ↓
框架封装
  ↓
Hook 和插件开发
  ↓
源码理解、面试表达、真实项目落地
```

真正掌握 pytest 的标志不是“知道很多装饰器”，而是能把测试写得稳定、清晰、可维护，并能随着项目变复杂继续扩展。
