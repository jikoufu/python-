# 04 参数化与数据驱动

## 是什么

参数化是用同一段测试逻辑运行多组测试数据。pytest 使用 `@pytest.mark.parametrize` 实现参数化。

```python
import pytest


@pytest.mark.parametrize(
    "a,b,expected",
    [
        (1, 2, 3),
        (0, 0, 0),
        (-1, 1, 0),
    ],
)
def test_add(a, b, expected):
    assert a + b == expected
```

## 为什么要用它

真实测试通常不是只测一组数据，而是要覆盖：

- 正常值
- 边界值
- 异常值
- 空值
- 多种业务状态

参数化可以减少重复代码，让一组规则覆盖多组输入。

## 怎么用

多参数参数化：

```python
@pytest.mark.parametrize(
    "username,password,expected_status",
    [
        ("admin", "123456", 200),
        ("admin", "wrong", 401),
        ("", "123456", 422),
    ],
)
def test_login(username, password, expected_status):
    assert isinstance(username, str)
```

给用例设置可读 ID：

```python
@pytest.mark.parametrize(
    "phone,expected",
    [
        pytest.param("13800138000", True, id="valid-phone"),
        pytest.param("", False, id="empty-phone"),
    ],
)
def test_phone(phone, expected):
    assert bool(phone) is expected
```

## 场景例子

场景：登录接口测试。

```python
import pytest
import requests


@pytest.mark.parametrize(
    "payload,expected_status",
    [
        ({"username": "admin", "password": "123456"}, 200),
        ({"username": "admin", "password": "wrong"}, 401),
        ({"username": "", "password": "123456"}, 422),
    ],
)
def test_login(base_url, payload, expected_status):
    response = requests.post(f"{base_url}/login", json=payload)
    assert response.status_code == expected_status
```

测试数据多时，可以从 JSON/YAML 读取，再喂给参数化。

## 大厂使用的例子

大型团队常用参数化维护测试矩阵：

- 多国家、多语言、多币种价格计算。
- 多会员等级、多优惠券组合。
- 多权限角色访问同一个接口。
- 多模型、多 Prompt 版本评测同一批问题。

这样做的好处是测试逻辑稳定，测试数据可以持续扩充。

## 本章小结

参数化让测试从“一条用例”变成“一组规则”。测试开发中要习惯把重复逻辑抽出来，把变化点沉淀为测试数据。
