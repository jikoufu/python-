# 01 RAG 测试核心概念

## 学习目标

学完这一节，你需要掌握：

- RAG 是什么。
- RAG 系统由哪些环节组成。
- RAG 测试为什么要拆链路。
- RAG 的常见失败类型。
- RAG 测试需要关注哪些指标。

## RAG 是什么

RAG 是 Retrieval-Augmented Generation，中文通常叫“检索增强生成”。

它的基本流程是：

```text
用户问题
    ↓
检索相关资料
    ↓
把资料放进 Prompt
    ↓
大模型基于资料回答
```

RAG 解决的问题：

- 大模型不知道企业私有知识。
- 大模型知识可能过时。
- 直接把所有文档塞进上下文成本太高。
- 需要回答能追溯来源。

## RAG 系统链路

一个典型 RAG 系统包含：

```text
文档采集
    ↓
文档清洗
    ↓
文档切分
    ↓
Embedding 向量化
    ↓
向量库入库
    ↓
用户提问
    ↓
问题改写或扩展
    ↓
检索 Top-k 文档片段
    ↓
重排序
    ↓
Prompt 拼接
    ↓
模型生成答案
    ↓
引用来源返回
```

测试时不要只测最后的答案，要分层排查。

## RAG 为什么难测

因为 RAG 有两个核心系统：

```text
检索系统
生成系统
```

如果回答错了，可能有很多原因：

- 正确文档没有被加载。
- 文档切分把关键信息切断了。
- Embedding 模型效果不好。
- Top-k 太小，没有召回正确片段。
- 检索到了正确片段，但排序太靠后。
- Prompt 没有要求模型基于上下文回答。
- 模型忽略上下文自己编答案。
- 引用来源错了。

所以 RAG 测试的关键是：定位错误发生在哪一层。

## RAG 常见失败类型

### 1. 文档加载失败

表现：

- 某些文件没进入知识库。
- PDF 表格、图片、标题解析错误。
- 文档元数据丢失。

测试点：

- 文件数量是否一致。
- 文档标题、路径、页码是否保留。
- 关键段落是否能在入库数据中找到。

### 2. 文档切分失败

表现：

- 一个完整概念被切成两段。
- chunk 太小，缺上下文。
- chunk 太大，检索不精准。
- overlap 不合理。

测试点：

- 关键答案是否在同一个 chunk 或相邻 chunk。
- 标题和正文是否分离。
- 表格是否被破坏。

### 3. 检索失败

表现：

- 问题明明在知识库里，却没检索到。
- 检索结果和问题无关。
- 关键词命中，但语义不相关。

测试点：

- Top-k 结果中是否包含正确文档。
- 第一条结果是否相关。
- 同义词、缩写、中文表达是否能召回。

### 4. 生成幻觉

表现：

- 上下文没有的信息，模型自己编。
- 回答和检索内容矛盾。
- 不知道的问题没有拒答。

测试点：

- 答案是否能在上下文中找到依据。
- 未知问题是否回答“不知道”或“资料中未提到”。

### 5. 引用错误

表现：

- 答案对，但引用错文件。
- 引用页码错误。
- 引用了不支持答案的片段。

测试点：

- 引用来源是否真的包含答案依据。
- 引用文件名、页码、段落是否准确。

## RAG 测试指标

常见指标包括：

### 检索侧指标

- Context Precision：检索到的上下文有多少是相关的。
- Context Recall：需要的上下文有没有被召回。
- Hit Rate：Top-k 中是否命中正确文档。
- MRR：正确文档排名是否靠前。

### 生成侧指标

- Faithfulness：答案是否忠实于检索上下文。
- Answer Relevancy：答案是否回答了问题。
- Answer Correctness：答案是否正确。
- Citation Accuracy：引用是否准确。

### 工程侧指标

- 响应时间
- Token 成本
- 检索耗时
- 生成耗时
- 失败率
- 超时率

## RAG 测试思维

一个 RAG 测试工程师看到错误回答时，要按这个顺序排查：

```text
1. 知识库里有没有答案？
2. 文档是否正确加载？
3. 文档是否正确切分？
4. 检索结果有没有召回正确片段？
5. 正确片段排名是否靠前？
6. Prompt 是否把上下文传给模型？
7. 模型是否基于上下文回答？
8. 引用是否指向正确来源？
```

## 本节重点

- RAG 测试必须拆成检索和生成两部分。
- 检索失败和生成失败要分开定位。
- RAG 不只测答案，还要测上下文和引用。
- Faithfulness、Context Precision、Context Recall 是核心指标。
- 未知问题拒答是 RAG 必测项。
