# AI 大模型测试工程师能力地图

## 一句话能力模型

AI 大模型测试工程师 = 测试工程基础 + Python 自动化 + 大模型应用理解 + AI 评估能力 + 安全风险意识 + 工程化落地能力。

## 能力分层

### L1：传统测试基本功

需要掌握：

- 测试用例设计
- 缺陷报告
- 测试计划
- 测试报告
- 回归测试
- 冒烟测试

判断标准：

- 能独立测试一个普通业务模块。
- 能写清楚测试点和缺陷复现步骤。

### L2：自动化测试能力

需要掌握：

- Python
- pytest
- requests / httpx
- Selenium
- fixture
- 参数化
- 测试报告
- CI 执行

判断标准：

- 能写接口自动化测试框架。
- 能写 Web 自动化测试。
- 能让测试在流水线里自动运行。

### L3：大模型应用理解

需要掌握：

- Prompt
- Token
- Temperature
- Function Calling
- Embedding
- RAG
- Agent
- LangChain / LlamaIndex 基础

判断标准：

- 能看懂大模型应用链路。
- 能知道问题出在 Prompt、模型、检索、工具还是业务代码。

### L4：AI 专项测试能力

需要掌握：

- Prompt 回归测试
- RAG 检索测试
- RAG 忠实性测试
- Agent 工具调用测试
- 输出格式测试
- 幻觉检测
- LLM-as-a-Judge

判断标准：

- 能设计 AI 测试集。
- 能批量评测模型输出。
- 能输出质量分析报告。

### L5：安全与风险测试能力

需要掌握：

- Prompt Injection
- Jailbreak
- 敏感信息泄露
- 越权访问
- RAG 数据污染
- 有害内容检测

判断标准：

- 能设计红队测试用例。
- 能发现常见 AI 安全风险。
- 能给出修复建议。

### L6：平台化与工程化能力

需要掌握：

- 测试数据管理
- 评测报告
- CI/CD
- 可观测性
- Token 成本统计
- Trace 分析
- 测试平台设计

判断标准：

- 能把 AI 测试做成长期可回归的工程体系。
- 能支撑模型、Prompt、RAG、Agent 的持续迭代。

## 学习优先级

如果时间有限，优先学：

1. Python + pytest + requests
2. HTTP 接口测试
3. 大模型基础
4. Prompt 测试
5. RAG 测试
6. Agent 测试
7. 评测报告
8. 安全测试
9. CI/CD

## 阶段性作品

建议做 4 个作品：

1. 接口自动化测试框架
2. AI 聊天接口测试集
3. RAG 问答评测报告
4. Agent 工具调用测试报告

这 4 个作品比只写学习笔记更能体现岗位能力。
