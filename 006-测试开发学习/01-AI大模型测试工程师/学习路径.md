# AI 大模型测试工程师学习路径

## 阶段 1：职业认知与能力模型

目标：先搞清楚这个岗位到底测什么。

学习内容：

- AI 大模型测试工程师是什么
- 和传统测试工程师的区别
- 和 AI 应用开发工程师的区别
- 大模型应用的风险类型
- 岗位能力地图

阶段成果：

- 能说清楚大模型测试的工作范围。
- 能列出 Prompt、RAG、Agent 的核心测试点。
- 知道自己的学习路线。

## 阶段 2：测试工程基础

目标：补齐传统测试基本功。

学习内容：

- 测试用例设计
- 等价类
- 边界值
- 场景法
- 因果图
- 缺陷生命周期
- 测试计划和测试报告

阶段成果：

- 能写规范测试用例。
- 能写缺陷报告。
- 能设计一个功能模块的测试点。

## 阶段 3：Python 与后端基础

目标：具备写自动化测试代码的能力。

学习内容：

- Python 基础
- 函数、类、模块
- 虚拟环境
- pip
- JSON / YAML
- 日志
- FastAPI 基础
- HTTP 协议

阶段成果：

- 能写 Python 脚本。
- 能调用接口。
- 能读写测试数据。
- 能理解后端接口怎么工作。

## 阶段 4：pytest 接口自动化测试

目标：能做后端接口自动化。

学习内容：

- pytest
- requests / httpx
- fixture
- 参数化
- Token 鉴权
- 接口依赖
- 测试报告
- FastAPI TestClient

阶段成果：

- 能写 GET / POST / PUT / DELETE 自动化用例。
- 能管理测试数据。
- 能自动断言状态码、JSON 和响应时间。

## 阶段 5：Selenium Web 自动化测试

目标：能测试 AI 应用前端交互。

学习内容：

- Selenium 基础
- 浏览器驱动
- 元素定位
- 显式等待
- 截图
- Page Object Model
- AI 聊天页面测试

阶段成果：

- 能自动打开页面、输入问题、点击发送。
- 能等待 AI 回复。
- 能断言页面状态。
- 能失败截图。

## 阶段 6：大模型基础

目标：理解大模型为什么难测。

学习内容：

- LLM
- Token
- Temperature
- 上下文窗口
- System Prompt
- 幻觉
- Function Calling
- Embedding
- 向量数据库

阶段成果：

- 能理解大模型输出为什么不稳定。
- 能知道哪些参数会影响回答。
- 能看懂模型调用请求和响应。

## 阶段 7：Prompt 测试

目标：能验证 Prompt 是否稳定、可控、可回归。

学习内容：

- Prompt 测试集
- 指令遵循
- 输出格式
- 边界输入
- 角色越狱
- Prompt 注入
- Prompt 版本对比

阶段成果：

- 能给 Prompt 建测试样本。
- 能比较两个 Prompt 版本效果。
- 能发现格式错误、拒答错误和指令不遵循问题。

## 阶段 8：RAG 测试

目标：能测试知识库问答是否可靠。

学习内容：

- 文档切分测试
- 检索召回测试
- 上下文相关性
- 答案忠实性
- 引用来源校验
- 未知问题拒答
- RAGAS 指标

阶段成果：

- 能区分是检索错了，还是生成错了。
- 能设计 RAG 测试集。
- 能判断回答是否基于知识库。

## 阶段 9：Agent 测试

目标：能测试带工具调用的大模型应用。

学习内容：

- 工具选择
- 工具参数
- 工具调用轨迹
- 多步骤任务
- 失败恢复
- 权限边界
- Agent 回归测试

阶段成果：

- 能检查 Agent 每一步是否合理。
- 能发现工具误调用、漏调用、重复调用。
- 能设计任务型测试用例。

## 阶段 10：评估指标与评测框架

目标：能把 AI 测试做成可重复评测。

学习内容：

- 规则断言
- 关键词断言
- JSON Schema
- 语义相似度
- LLM-as-a-Judge
- RAGAS
- DeepEval
- Promptfoo
- 评测报告

阶段成果：

- 能批量运行 AI 测试集。
- 能输出通过率和失败原因。
- 能沉淀一套评测指标。

## 阶段 11：安全与红队测试

目标：能发现 AI 应用安全风险。

学习内容：

- Prompt Injection
- Jailbreak
- 数据泄露
- 越权访问
- 有害输出
- 敏感信息检测
- 安全测试集

阶段成果：

- 能写 AI 安全测试用例。
- 能验证系统提示词是否容易泄露。
- 能验证 RAG 是否会被恶意文档污染。

## 阶段 12：性能、成本与可观测性

目标：能从工程角度评估 AI 应用是否可上线。

学习内容：

- 响应时间
- 并发测试
- 超时和重试
- Token 消耗
- 成本统计
- 日志
- Trace
- LangSmith / Phoenix

阶段成果：

- 能统计接口延迟和 Token 成本。
- 能定位慢请求。
- 能追踪一次 AI 调用完整链路。

## 阶段 13：CI/CD 与测试平台

目标：把 AI 测试接入持续集成。

学习内容：

- Git
- GitHub Actions / GitLab CI
- Jenkins
- 测试报告
- 测试数据版本管理
- 定时回归
- 评测结果归档

阶段成果：

- 能让测试自动运行。
- 能在 Prompt 或代码变更后自动回归。
- 能保存历史评测结果。

## 阶段 14：面试与作品集

目标：形成能展示的项目能力。

推荐作品：

- AI 聊天接口自动化测试框架
- Prompt 回归测试工具
- RAG 问答评测集
- Agent 工具调用测试报告
- AI 安全红队测试用例集

最终成果：

- 一份学习笔记
- 一个自动化测试项目
- 一份 AI 应用测试报告
- 一份面试题整理
- 一套可展示的 Git 项目结构
