# 02 主流评测框架选型

## 学习目标

这一节学习常见评测框架怎么选。

重点了解：

- RAGAS
- DeepEval
- Promptfoo

## RAGAS

RAGAS 主要用于评估 RAG 应用。

根据官方文档，Ragas 提供多类评估指标，可用于 RAG 和 Agentic workflows，其中 RAG 常见指标包括：

- Context Precision
- Context Recall
- Faithfulness
- Answer Relevancy

适合场景：

- 知识库问答评估
- 检索质量评估
- 答案忠实性评估
- RAG 方案对比

你可以用它回答：

```text
检索结果相关吗？
该召回的资料召回了吗？
答案是否基于上下文？
回答是否切题？
```

## DeepEval

DeepEval 是偏 Python 生态的 LLM 评测框架。

适合场景：

- Python 项目里写评测用例
- 类 pytest 的评测流程
- 通用 LLM 输出评估
- RAG 评估
- 自定义指标
- 回归测试

适合你这种学习路线，因为你已经在学 pytest 和接口自动化。

你可以用它回答：

```text
这个回答是否相关？
是否幻觉？
是否完成任务？
是否符合预期？
```

## Promptfoo

Promptfoo 是配置化的 LLM eval 和红队测试工具。

根据官方介绍，它是 open-source CLI 和 library，可用于评估和红队测试 LLM 应用，并支持用声明式测试用例定义 eval。

适合场景：

- Prompt A/B 测试
- 多模型对比
- 配置化测试集
- 红队测试
- CI 中快速跑 eval

你可以用它回答：

```text
哪个 Prompt 效果更好？
哪个模型更稳定？
这个应用能不能抵抗 Prompt 注入？
CI 中这次 Prompt 修改有没有回退？
```

## 简单选型

| 场景 | 推荐 |
|------|------|
| RAG 质量评估 | RAGAS |
| Python 自动化评测 | DeepEval |
| Prompt 对比和红队 | Promptfoo |
| 简单接口断言 | pytest + requests |
| 自定义业务评估 | pytest + 自定义规则 + LLM Judge |

## 学习建议

先不要同时深入所有框架。

推荐顺序：

1. pytest + 自定义规则评估
2. RAGAS
3. DeepEval
4. Promptfoo

原因：

- pytest 是基础。
- RAGAS 对 RAG 测试最直接。
- DeepEval 适合做 Python 项目化。
- Promptfoo 适合 Prompt 对比和安全红队。

## 本节重点

- RAGAS 偏 RAG 指标。
- DeepEval 偏 Python 评测框架。
- Promptfoo 偏配置化 eval、Prompt 对比和红队。
- 框架只是工具，测试集和评估标准才是核心资产。
