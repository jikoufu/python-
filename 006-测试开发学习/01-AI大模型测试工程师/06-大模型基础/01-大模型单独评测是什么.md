# 01 大模型单独评测是什么

## 学习目标

学完这一节，你需要掌握：

- 什么叫单独对大模型评测。
- 大模型评测和 AI 应用测试有什么区别。
- 大模型评测从哪些维度做。
- 面试中如何回答“你有没有单独做过大模型评测”。

## 什么叫单独对大模型评测

单独对大模型评测，意思是先不看具体业务系统，只评估模型本身能力。

也就是说，不看：

- RAG 检索链路
- Agent 工具调用
- 前端页面
- 后端业务接口
- 企业知识库

而是准备一批统一测试问题，直接给模型回答，然后比较模型能力。

例如同一批问题分别给这些模型：

```text
GPT
Claude
Qwen
DeepSeek
Llama
```

然后比较它们的回答质量、稳定性、安全性、速度和成本。

## 大模型评测 vs AI 应用测试

### 大模型评测

测试对象：

```text
模型本身
```

关注：

```text
模型会不会答
模型答得准不准
模型是否遵循指令
模型是否容易幻觉
模型能不能稳定输出格式
模型是否安全拒答
```

### AI 应用测试

测试对象：

```text
模型 + Prompt + RAG + Agent + 后端接口 + 前端展示 + 业务规则
```

关注：

```text
业务系统整体是否可用
RAG 是否检索正确
Agent 是否调用正确工具
Prompt 是否稳定
接口是否正常
前端是否展示正确
```

## 举例说明

问题：

```text
公司年假规则是什么？
```

如果是大模型评测：

```text
看模型是否理解问题、表达清楚、是否会乱编。
```

如果是 RAG 应用测试：

```text
还要看是否检索到员工手册、回答是否基于上下文、引用是否正确。
```

所以应用测试比模型评测链路更长。

## 大模型评测维度

### 1. 准确性

回答是否正确。

示例：

```text
问题：Python 中 list 和 tuple 有什么区别？
```

评估：

- 是否说清可变和不可变。
- 是否举例正确。

### 2. 指令遵循

模型是否按要求回答。

示例：

```text
请用 3 条 bullet 回答。
```

评估：

- 是否正好 3 条。
- 是否没有额外废话。

### 3. 格式稳定性

模型能否稳定输出指定格式。

示例：

```text
请只输出 JSON，字段包括 title、steps、expected_result。
```

评估：

- JSON 是否合法。
- 字段是否完整。
- 类型是否正确。
- 是否夹带自然语言解释。

### 4. 幻觉率

模型是否编造不存在的信息。

测试方式：

- 问一些模型不知道的问题。
- 问一些资料中不存在的问题。
- 看模型是否承认不知道。

### 5. 安全性

模型是否能拒绝不安全请求。

测试内容：

- 隐私信息
- 危险操作
- 违法内容
- 越狱提示
- Prompt 注入

### 6. 中文能力

对于国内或中文业务场景，需要测试：

- 中文理解
- 中文表达
- 中文术语
- 中英混合
- 方言或口语

### 7. 推理能力

测试多步骤推理：

- 数学题
- 逻辑题
- 规则推理
- 多条件判断
- 测试用例设计推理

### 8. 代码能力

如果岗位涉及代码测试或开发，需要评估：

- 代码生成
- 代码解释
- Bug 分析
- 单元测试生成
- 接口自动化脚本生成

### 9. 多轮对话能力

测试模型是否能记住当前对话上下文。

注意：

```text
这通常还是上下文能力，不等于模型本身有长期记忆。
```

### 10. 延迟和成本

工程上还要看：

- 首 token 延迟
- 总响应时间
- 输入 token
- 输出 token
- 单次调用成本
- 错误率
- 超时率

## 大模型评测方法

### 规则评估

适合：

- JSON 是否合法
- 是否包含关键词
- 字段是否齐全
- 响应时间是否小于阈值

### 人工评估

适合：

- 业务正确性
- 表达质量
- 复杂推理
- 主观体验

### LLM-as-a-Judge

使用另一个模型当评委。

适合：

- 回答相关性
- 忠实性
- 完整性
- 安全性

注意：

```text
LLM Judge 也会误判，高风险场景要人工抽查。
```

### 评测框架

常见工具：

- DeepEval
- Promptfoo
- RAGAS
- 自研 pytest 测试集

## 面试回答模板

如果面试官问：

```text
你有没有单独对大模型做过评测？
```

如果你没有实际做过，可以这样回答：

```text
我之前更多做的是大模型应用侧测试，比如 Prompt、RAG、Agent 和接口自动化。单独的大模型评测我理解是脱离具体业务链路，用统一测试集评估模型本身能力，包括准确性、指令遵循、格式稳定性、幻觉率、安全性、中文能力、延迟和 Token 成本等。评测方法可以结合规则断言、人工评估、LLM-as-a-Judge，以及 DeepEval、Promptfoo 这类工具。
```

这个回答的重点是：

- 不虚构经历。
- 说明你知道模型评测和应用测试的区别。
- 说明你知道怎么做。

## 本节重点

- 单独大模型评测测的是模型本身。
- AI 应用测试测的是完整业务链路。
- 大模型评测常看准确性、指令遵循、格式稳定性、幻觉、安全、延迟和成本。
- 没做过可以承认，但要说清评测思路。
