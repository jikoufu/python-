# 01 Agent 测试核心概念

## 学习目标

学完这一节，你需要掌握：

- Agent 是什么。
- Agent 和普通聊天机器人的区别。
- Agent 测试为什么要看执行轨迹。
- Agent 常见失败类型。

## Agent 是什么

Agent 可以理解为：

```text
大模型 + 工具 + 规划 + 执行 + 观察
```

普通聊天机器人主要是回答。

Agent 可以做动作，例如：

- 查询数据库
- 调用搜索接口
- 创建工单
- 发送邮件
- 读取文件
- 调用计算器
- 操作浏览器

## Agent 执行流程

典型流程：

```text
用户任务
    ↓
模型理解任务
    ↓
制定计划
    ↓
选择工具
    ↓
生成工具参数
    ↓
调用工具
    ↓
观察工具结果
    ↓
继续调用工具或生成最终答案
```

## Agent 为什么难测

因为 Agent 的流程不是固定的。

同一个任务，Agent 可能：

- 选择不同工具
- 调用不同次数
- 参数写法不同
- 中途失败后选择不同恢复策略
- 最终答案类似但过程不合规

所以只看最终答案是不够的。

## Agent 测试必须看什么

### 1. 工具选择

问题：

```text
查询北京天气。
```

期望：

```text
调用 get_weather 工具。
```

如果 Agent 调用了 `send_email`，就是严重错误。

### 2. 工具参数

期望：

```json
{
  "city": "北京"
}
```

错误：

```json
{
  "city": "上海"
}
```

### 3. 调用顺序

例如订票任务：

```text
查询车次 → 检查余票 → 创建订单
```

如果顺序变成：

```text
创建订单 → 查询车次
```

就是流程错误。

### 4. 权限边界

普通用户不能调用管理员工具。

例如：

```text
普通用户请求删除所有订单。
```

期望：

```text
拒绝或提示无权限。
```

不能真的调用删除工具。

### 5. 失败恢复

工具调用失败时，Agent 应该：

- 重试
- 换工具
- 向用户说明失败
- 请求补充信息

不能无限循环，也不能编造成功。

## Agent 常见失败类型

| 失败类型 | 说明 |
|----------|------|
| 工具误选 | 选择了不该用的工具 |
| 工具漏调 | 应该调用工具却直接回答 |
| 参数错误 | 工具参数和用户意图不一致 |
| 顺序错误 | 多步骤任务顺序不合理 |
| 无限循环 | 反复调用同一个工具 |
| 越权调用 | 调用了无权限工具 |
| 编造结果 | 工具失败却说成功 |
| 忽略观察 | 工具返回错误，Agent 没处理 |

## Agent 测试分层

```text
任务理解测试
工具选择测试
工具参数测试
工具执行测试
轨迹合理性测试
最终答案测试
权限安全测试
失败恢复测试
```

## 本节重点

- Agent 测试不能只看最终答案。
- 工具选择和参数是核心测试点。
- 多步骤任务要看调用顺序。
- 权限和失败恢复是高风险点。
- 执行轨迹是 Agent 测试的关键证据。
