## 第一部分：学习路线（4 阶段）

### 阶段 0：打地基（先决条件，1 周）

测 Agent 之前，得先**会用** Agent，否则测无从下手。

```text
要会的：
- 能用 Claude/OpenAI 的 Function Calling 跑通一次工具调用
- 理解 Agent 的 ReAct 循环（思考→行动→观察→再思考）
- 会用一个 Agent 框架跑个 demo（LangChain 或 LangGraph 二选一）
```

> 对应本仓库：`003-ai学习/09-AI-Agent开发/01-理解AI-Agent.md`（先把 Agent 原理过一遍）

**阶段成果**：自己写一个带 2~3 个工具的小 Agent，能跑通"用户提问→调工具→回答"。

---

### 阶段 1：Tool Call 测试（2 周，重点）

工具调用是 Agent 测试里最具体、最好上手的部分，先啃它。

```text
学习内容：
1. 工具调用的 5 个环节、7 个测试维度
2. 怎么对工具名、参数做结构化断言
3. 重点攻克：参数幻觉、该不该调用、错误处理、越权
4. 把用例组织成数据集，批量跑出准确率

对应本仓库文档：
→ 05-ToolCall测试详解.md（先读这篇，把维度和断言代码吃透）
→ 02-Agent测试用例设计.md（用例设计方法）
```

**阶段成果**：写出一组 pytest 工具调用测试用例，覆盖"选对工具/参数正确/参数幻觉/越权拒绝/错误处理"，能跑出通过率。

---

### 阶段 2：Agent 整体测试（2 周）

从"测一次调用"升级到"测一整条执行轨迹"。

```text
学习内容：
1. 执行轨迹（Trace）是什么，为什么 Agent 测试必须看轨迹
2. 多步任务的顺序、漏步、冗余、死循环怎么测
3. 任务完成度怎么评（不只看最终答案，要看过程合不合规）
4. Memory / 多轮上下文怎么测
5. 失败恢复、权限边界

对应本仓库文档：
→ 01-Agent测试核心概念.md（轨迹、失败类型、测试分层）
→ 02-Agent测试用例设计.md
→ 04-Agent的Memory机制.md
```

**阶段成果**：给一个多步 Agent（如订票/查询助手）设计一份完整测试方案，覆盖轨迹、顺序、Memory、失败恢复。

---

### 阶段 3：自动化评测 + 工程化（2 周）

把零散用例变成能持续运行的评测体系——这是简历亮点。

```text
学习内容：
1. LLM-as-a-Judge：用模型评判 Agent 表现（注意避免位置偏见）
2. 评测数据集的组织和版本管理
3. 把评测接进 CI（每次改 prompt/模型自动回归）
4. 输出评测报告

对应本仓库文档：
→ 03-Agent自动化评测流程.md
→ 10-评估指标与评测框架/（整个模块）
```

**阶段成果**：一个能一键运行、输出报告的 Agent 评测脚本，最好接进 GitHub Actions。

---

### 路线总览图

```text
阶段0 会用Agent ──→ 阶段1 ToolCall测试 ──→ 阶段2 Agent整体测试 ──→ 阶段3 自动化评测
  1周               2周(重点)            2周                  2周
  跑通demo          结构化断言           看轨迹               进CI出报告
```

总计约 **7 周**，每天 1~1.5 小时。

## 第二部分：精选学习资料

不贪多，每类只给最值得看的。

### 必读（理解原理）

| 资料 | 为什么看 |
|------|---------|
| **Anthropic《Building Effective Agents》** | 讲清 Workflow vs Agent、各种 Agent 模式，业界公认最佳入门 |
| **Anthropic Tool Use 官方文档** | Function Calling 怎么定义工具、怎么写 schema，测试前必懂 |
| **ReAct 论文（ReAct: Reasoning + Acting）** | Agent"思考-行动-观察"循环的源头，理解轨迹概念 |

### 框架文档（动手用）

| 资料 | 用途 |
|------|------|
| **LangChain / LangGraph 文档** | 主流 Agent 框架，跑 demo、理解 Agent 怎么搭 |
| **LangSmith** | LangChain 的调试/追踪/评测平台，能看 Agent 执行轨迹，**对测试极有用** |
| **pytest 文档** | 你的测试用例载体，重点看 fixture、parametrize、mock |

### 评测工具（阶段3 用）

| 资料 | 用途 |
|------|------|
| **DeepEval** | 专门做 LLM/Agent 评测的框架，有现成的工具调用、任务完成度指标 |
| **RAGAS** | 偏 RAG 评测，但 LLM-as-a-Judge 思路通用 |
| **Promptfoo** | 轻量的 prompt/Agent 评测，配置化，容易接 CI |

### 学习方式建议

```text
原理类（论文/Anthropic 文档）：读 1 遍，做笔记，不用抠细节
框架类（LangGraph/pytest）：边看边敲，跑通比读完更重要
评测工具：用项目驱动，做练手项目时顺手学，别孤立地学
```

> 提醒：**别陷入"资料收集癖"**。上面每类挑 1 个先用起来，比收藏 50 个教程有用得多。

## 第三部分：练手项目（重点，补你的项目短板）

这是这份文档的核心。4 个项目由易到难，每个都能写进简历。

### 项目1：天气查询 Agent 的工具调用测试集（入门）

```text
做什么：
- 搭一个带 3 个工具的简单 Agent（查天气/查时间/计算器）
- 写一组 pytest 用例，测：工具选对没、参数对不对、闲聊不误触发、
  参数幻觉、无权限工具被拒
- 跑出通过率

练到的能力：Tool Call 结构化断言、用例设计
简历可写：「设计 Agent 工具调用自动化测试集，覆盖 N 个维度，
          发现参数幻觉等 X 类问题」
```

### 项目2：订票/订单 Agent 的轨迹测试（进阶）

```text
做什么：
- 一个多步 Agent（查车次→查余票→下单），故意埋几个 bug
- 测调用顺序、漏步骤、死循环、工具失败后的恢复
- 用断言验证整条执行轨迹

练到的能力：轨迹测试、多步任务、失败恢复
简历可写：「针对多步骤 Agent 设计轨迹级测试，覆盖调用顺序、
          异常恢复，定位 X 个流程缺陷」
```

### 项目3：LLM-as-a-Judge 评测脚本（拔高）

```text
做什么：
- 准备一个评测数据集（20~50 条任务 + 期望表现）
- 写一个用裁判模型给 Agent 回答打分的脚本
- 处理位置偏见（交换顺序再评）、输出结构化分数
- 生成一份 Markdown/HTML 评测报告

练到的能力：自动化评测、LLM-as-a-Judge
简历可写：「构建 Agent 自动化评测体系，基于 LLM-as-a-Judge
          实现批量打分与报告生成」
```

### 项目4：接进 CI 的回归评测（综合，最值钱）

```text
做什么：
- 把项目1~3 的用例整理成评测套件
- 接进 GitHub Actions：每次改 prompt/换模型自动跑评测
- 评分低于阈值就让流水线失败（质量门禁）

练到的能力：测试工程化、CI/CD、质量门禁
简历可写：「搭建 Agent 质量回归流水线，prompt/模型变更自动评测，
          通过质量门禁拦截效果回退」
```

### 项目推进建议

```text
1. 按 1→2→3→4 顺序做，每个项目对应学习路线一个阶段
2. 每个项目都建 git 仓库，commit 记录就是你的学习轨迹
3. 给每个项目写一个 README：背景、测了什么、发现什么问题、结论
   （面试时这个 README 就是你的作品讲稿）
4. 做完 4 个，你就有一条完整的"会测 Agent"的项目线
```
