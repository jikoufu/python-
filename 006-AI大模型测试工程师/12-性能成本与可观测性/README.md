# 12 性能、成本与可观测性

性能测试是测试工程师的核心硬技能，也是面试高频区。这一章以 **JMeter** 为主线，讲清楚怎么做压测、怎么分析性能瓶颈、怎么测微服务，并配套大量面试题。

对 AI 大模型测试工程师来说，性能测试还有一层特殊性：大模型接口又慢又贵，性能和成本必须一起看（见本章最后一节）。

## 本章你要学会什么

- 说清性能测试的核心概念和指标（TPS、RT、并发、吞吐量）
- 用 JMeter 设计并执行压测脚本
- 看懂压测结果，定位性能瓶颈（CPU/内存/IO/数据库/锁/连接池）
- 理解微服务架构下性能测试的特殊性
- 应对性能测试的常见面试题
- 把性能测试和大模型的成本、可观测性结合起来

## 核心结论

```text
压测不是为了"压出一个漂亮的 TPS 数字"，
而是为了找到系统的瓶颈在哪、上限是多少、什么时候会崩。

会发压的人很多，会分析瓶颈的人才值钱。
```
