# 01 评估指标核心概念

## 学习目标

学完这一节，你需要掌握：

- AI 评估为什么难。
- 规则评估、人工评估、LLM-as-a-Judge 的区别。
- 常见评估指标。
- 如何选择指标。

## 为什么 AI 评估难

传统接口测试可以这样断言：

```python
assert response.status_code == 200
assert response.json()["code"] == 0
```

但 AI 回答可能有多种正确表达。

例如：

```text
FastAPI 是一个 Python Web 框架。
```

和：

```text
FastAPI 是用于构建 API 的高性能 Python 后端框架。
```

文字不同，但都可能正确。

所以 AI 评估不能只靠完全相等。

## 三类评估方式

### 1. 规则评估

用代码规则判断。

例如：

- 回答不能为空。
- JSON 格式合法。
- 必须包含某些字段。
- 状态码必须是 200。
- 响应时间小于 3 秒。

优点：

- 快
- 便宜
- 稳定

缺点：

- 不能完整判断语义质量。

### 2. 人工评估

由人打分。

适合：

- 主观质量
- 业务准确性
- 高风险场景
- 评估标准制定初期

缺点：

- 慢
- 成本高
- 不容易大规模回归

### 3. LLM-as-a-Judge

用另一个大模型当评委。

适合：

- 回答相关性
- 忠实性
- 有害内容
- 任务完成度
- 多维度评分

风险：

- 评委模型也会错。
- 评分可能不稳定。
- 成本更高。
- 需要抽样人工复核。

## 常见指标

### 通用回答指标

| 指标 | 含义 |
|------|------|
| Answer Relevancy | 是否回答了用户问题 |
| Correctness | 答案是否正确 |
| Completeness | 答案是否完整 |
| Conciseness | 是否简洁 |
| Format Validity | 输出格式是否正确 |
| Toxicity | 是否有害 |
| Bias | 是否有偏见 |

### RAG 指标

| 指标 | 含义 |
|------|------|
| Context Precision | 检索内容有多少是相关的 |
| Context Recall | 必要内容是否被召回 |
| Faithfulness | 答案是否忠实于上下文 |
| Citation Accuracy | 引用是否准确 |
| Refusal Accuracy | 不知道时是否拒答 |

### Agent 指标

| 指标 | 含义 |
|------|------|
| Tool Correctness | 工具选择是否正确 |
| Argument Correctness | 工具参数是否正确 |
| Task Completion | 任务是否完成 |
| Step Efficiency | 步骤是否冗余 |
| Permission Safety | 是否越权 |
| Recovery Ability | 工具失败后是否恢复 |

## 指标选择原则

不要一开始就追求很多指标。

建议先按场景选：

```text
普通聊天：相关性、正确性、安全性
结构化输出：格式合法、字段完整、类型正确
RAG：检索命中、忠实性、引用准确
Agent：工具选择、参数正确、任务完成
生产系统：延迟、成本、错误率、稳定性
```

## 本节重点

- AI 评估要结合规则、人工和 LLM Judge。
- 规则评估适合基础质量门禁。
- LLM Judge 适合语义质量评估，但不能完全迷信。
- RAG 和 Agent 要有专门指标。
- 指标要服务定位问题，而不是只追求一个总分。
