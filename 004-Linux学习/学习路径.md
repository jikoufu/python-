# Linux 学习路径步骤

## 阶段 1：系统基础与文件操作

目标：在 Linux 上能自如地导航、操作文件，不再依赖图形界面。

### 环境准备

**Windows 用户推荐 WSL2：**

```powershell
# 以管理员身份运行 PowerShell
wsl --install
# 安装完重启，选择 Ubuntu 22.04
```

### 核心命令

**目录导航**

```bash
pwd              # 当前所在目录
ls -la           # 列出所有文件（含隐藏文件和权限）
cd /etc          # 进入 /etc 目录
cd ~             # 回到家目录
cd -             # 回到上一个目录
```

**文件操作**

```bash
mkdir -p projects/demo    # 递归创建目录
touch app.py              # 创建空文件
cp -r src/ dst/           # 递归复制目录
mv old.py new.py          # 重命名/移动
rm -rf temp/              # 强制删除（谨慎！）
ln -s /path/to/file link  # 创建软链接
```

**文件查看**

```bash
cat file.txt              # 输出全部内容
less file.txt             # 分页查看（q 退出）
head -20 file.txt         # 查看前 20 行
tail -f app.log           # 实时追踪日志（最常用）
grep -rn "error" logs/    # 递归搜索关键字并显示行号
```

**文本处理三剑客**

```bash
# grep：搜索
grep -i "ERROR" app.log           # 不区分大小写
grep -v "DEBUG" app.log           # 反向匹配（排除 DEBUG）

# awk：列处理
awk '{print $1, $3}' access.log   # 输出第 1、3 列
awk -F: '{print $1}' /etc/passwd  # 用 : 分割，输出第 1 列

# sed：流编辑
sed 's/old/new/g' file.txt        # 替换所有 old 为 new
sed -n '10,20p' file.txt          # 输出第 10~20 行
```

**文件查找**

```bash
find / -name "nginx.conf" 2>/dev/null    # 全局查找文件名
find . -name "*.py" -mtime -1            # 最近 1 天修改的 .py 文件
find . -size +100M                       # 大于 100MB 的文件
which python3                            # 查找命令的路径
```

**压缩解压**

```bash
tar -czf archive.tar.gz dir/    # 压缩
tar -xzf archive.tar.gz         # 解压
zip -r archive.zip dir/         # zip 压缩
unzip archive.zip               # zip 解压
```

### 必须理解的目录结构

```text
/
├── etc/        配置文件（nginx.conf、hosts、passwd 都在这）
├── var/        可变数据（日志 /var/log、数据库文件）
├── opt/        第三方软件安装目录
├── home/       用户家目录（/home/ubuntu）
├── root/       root 用户的家目录
├── tmp/        临时文件（重启清空）
├── proc/       内核和进程信息（虚拟文件系统）
├── usr/        用户程序（/usr/bin、/usr/local）
└── srv/        服务数据
```

### 阶段成果

- 能在 Linux 上完成任意文件的增删改查。
- 能用 `grep`、`tail -f` 排查日志。
- 清楚知道配置文件在哪里。

## 阶段 2：用户与权限管理

目标：理解 Linux 权限模型，能安全配置服务器访问控制。

### 权限理解

```bash
ls -la
# -rw-r--r-- 1 ubuntu ubuntu 1234 Jan 1 10:00 app.py
#  ↑owner↑  ↑group↑  ↑others↑
# r=4 读  w=2 写  x=1 执行
```

**权限计算**

```bash
chmod 755 script.sh     # rwxr-xr-x：所有人可读执行，只有 owner 可写
chmod 644 config.json   # rw-r--r--：所有人可读，只有 owner 可写
chmod +x run.sh         # 给所有人加执行权限
chown ubuntu:ubuntu file.txt  # 修改所有者和组
```

### 用户管理

```bash
# 创建用户
useradd -m -s /bin/bash deploy
passwd deploy

# 加入 sudo 组（Ubuntu）
usermod -aG sudo deploy

# 切换用户
su - deploy

# 查看当前用户身份
whoami
id
```

### SSH 密钥登录（重要）

```bash
# 本地生成密钥对
ssh-keygen -t ed25519 -C "your_email@example.com"

# 上传公钥到服务器
ssh-copy-id -i ~/.ssh/id_ed25519.pub user@server_ip

# 服务器端：禁用密码登录（安全加固）
# 编辑 /etc/ssh/sshd_config
PasswordAuthentication no
PermitRootLogin no

sudo systemctl restart sshd
```

### sudo 配置

```bash
# 编辑 sudoers（必须用 visudo）
sudo visudo

# 允许 deploy 用户无密码执行 systemctl
deploy ALL=(ALL) NOPASSWD: /bin/systemctl
```

### 阶段成果

- 理解 `rwx` 权限的含义和数字表示。
- 能创建专用服务账号，不用 root 运行应用。
- 能配置 SSH 密钥登录，禁用密码登录。

## 阶段 3：进程与服务管理

目标：能管理运行中的进程，能把 Python 应用注册为系统服务。

### 进程查看

```bash
ps aux                  # 所有进程
ps aux | grep python    # 过滤 Python 进程
top                     # 实时进程监控（按 CPU 排序）
htop                    # 更友好的 top（需安装 sudo apt install htop）

# 查看某个端口被哪个进程占用
ss -tlnp | grep 8000
lsof -i :8000
```

### 进程控制

```bash
kill 1234           # 发送 SIGTERM（优雅关闭）
kill -9 1234        # 发送 SIGKILL（强制杀死）
pkill -f "uvicorn"  # 按进程名杀死

# 后台运行
nohup python app.py > app.log 2>&1 &
# & 放后台，nohup 防止终端关闭后进程退出
```

### systemd 服务管理（重点）

```bash
# 常用命令
systemctl start nginx       # 启动
systemctl stop nginx        # 停止
systemctl restart nginx     # 重启
systemctl reload nginx      # 热重载配置
systemctl enable nginx      # 开机自启
systemctl disable nginx     # 取消开机自启
systemctl status nginx      # 查看状态

# 查看日志
journalctl -u nginx -f      # 实时追踪 nginx 日志
journalctl -u nginx -n 100  # 查看最近 100 行
```

### 编写 systemd service 文件

将 FastAPI 应用注册为系统服务：

```ini
# /etc/systemd/system/myapp.service

[Unit]
Description=My FastAPI Application
After=network.target

[Service]
Type=exec
User=deploy
WorkingDirectory=/opt/myapp
ExecStart=/opt/myapp/venv/bin/gunicorn main:app -w 4 -k uvicorn.workers.UvicornWorker -b 0.0.0.0:8000
Restart=always
RestartSec=5
StandardOutput=journal
StandardError=journal

[Install]
WantedBy=multi-user.target
```

```bash
sudo systemctl daemon-reload
sudo systemctl enable myapp
sudo systemctl start myapp
sudo systemctl status myapp
```

### 定时任务 crontab

```bash
crontab -e   # 编辑当前用户的定时任务

# 格式：分 时 日 月 周 命令
# 每天凌晨 2 点执行备份脚本
0 2 * * * /opt/scripts/backup.sh >> /var/log/backup.log 2>&1

# 每 5 分钟检查一次服务
*/5 * * * * systemctl is-active myapp || systemctl restart myapp
```

### 阶段成果

- 能查看系统中所有进程，找到占用端口的进程。
- 能把 Python 应用注册为 systemd 服务，实现开机自启和崩溃重启。
- 能用 crontab 设置定时任务。

## 阶段 4：网络与防火墙

目标：理解服务器网络配置，能安全地开放和关闭端口。

### 网络状态查看

```bash
ip addr             # 查看网卡和 IP
ip route            # 查看路由表
ping 8.8.8.8        # 测试网络连通性
traceroute google.com  # 追踪路由路径

# 查看所有监听端口
ss -tlnp
netstat -tlnp       # 老版本系统用

# 查看连接状态
ss -s               # 连接统计
ss -antp            # 所有 TCP 连接
```

### 防火墙配置

**Ubuntu（ufw）**

```bash
sudo ufw enable                    # 启用防火墙
sudo ufw status verbose            # 查看规则

sudo ufw allow 22                  # 允许 SSH
sudo ufw allow 80                  # 允许 HTTP
sudo ufw allow 443                 # 允许 HTTPS
sudo ufw allow 8000                # 允许应用端口
sudo ufw deny 3306                 # 禁止外部访问 MySQL（只能本机访问）

sudo ufw delete allow 8000         # 删除规则
```

**CentOS（firewalld）**

```bash
sudo firewall-cmd --state
sudo firewall-cmd --add-port=80/tcp --permanent
sudo firewall-cmd --reload
```

### curl 实战

```bash
# 测试接口
curl http://localhost:8000/health

# POST 请求
curl -X POST http://localhost:8000/api/users \
  -H "Content-Type: application/json" \
  -d '{"name": "张三", "email": "zs@example.com"}'

# 带 Token
curl -H "Authorization: Bearer xxx" http://localhost:8000/api/me

# 下载文件
curl -O https://example.com/file.tar.gz

# 查看响应头
curl -I http://example.com
```

### SSH 端口转发

```bash
# 本地端口转发：把远程服务器的 3306 映射到本地 3307
# 在本地访问 127.0.0.1:3307 等于访问远程 MySQL
ssh -L 3307:127.0.0.1:3306 user@server_ip -N

# 动态端口转发（SOCKS 代理）
ssh -D 1080 user@server_ip -N
```

### 阶段成果

- 能查看服务器监听了哪些端口。
- 能用 ufw/firewalld 配置防火墙规则。
- 能用 SSH 端口转发安全访问服务器内部服务。

## 阶段 5：Shell 脚本

目标：能编写实用的自动化脚本，减少重复操作。

### Bash 基础语法

```bash
#!/bin/bash
set -e          # 遇到错误立即退出
set -o pipefail # 管道中任何命令失败都算失败

# 变量
name="World"
echo "Hello, $name"
echo "当前目录：$(pwd)"         # 命令替换

# 特殊变量
echo "脚本名：$0"
echo "第一个参数：$1"
echo "所有参数：$@"
echo "参数个数：$#"
echo "上一条命令的退出码：$?"
```

### 条件与循环

```bash
# if 条件
if [ -f "/etc/nginx/nginx.conf" ]; then
    echo "Nginx 配置文件存在"
elif [ -d "/etc/nginx" ]; then
    echo "Nginx 目录存在但没有配置文件"
else
    echo "Nginx 未安装"
fi

# 字符串比较
if [ "$ENV" = "production" ]; then
    echo "生产环境"
fi

# for 循环
for service in nginx mysql redis; do
    systemctl is-active $service && echo "$service 运行中" || echo "$service 未运行"
done

# while 循环（等待服务启动）
while ! curl -s http://localhost:8000/health > /dev/null; do
    echo "等待应用启动..."
    sleep 2
done
echo "应用已就绪"
```

### 实用脚本模板

**部署脚本**

```bash
#!/bin/bash
set -e

APP_DIR="/opt/myapp"
VENV_DIR="$APP_DIR/venv"
LOG_FILE="/var/log/deploy.log"

log() {
    echo "[$(date '+%Y-%m-%d %H:%M:%S')] $1" | tee -a "$LOG_FILE"
}

log "开始部署..."

cd "$APP_DIR"
git pull origin main
log "代码拉取完成"

source "$VENV_DIR/bin/activate"
pip install -r requirements.txt -q
log "依赖安装完成"

sudo systemctl restart myapp
sleep 3

if systemctl is-active myapp; then
    log "部署成功"
else
    log "ERROR: 服务启动失败"
    exit 1
fi
```

**数据库备份脚本**

```bash
#!/bin/bash
set -e

DB_NAME="mydb"
BACKUP_DIR="/backup/mysql"
DATE=$(date +%Y%m%d_%H%M%S)
BACKUP_FILE="$BACKUP_DIR/${DB_NAME}_${DATE}.sql.gz"

mkdir -p "$BACKUP_DIR"
mysqldump -u root -p"$MYSQL_PASSWORD" "$DB_NAME" | gzip > "$BACKUP_FILE"

# 只保留最近 7 天的备份
find "$BACKUP_DIR" -name "*.sql.gz" -mtime +7 -delete

echo "备份完成：$BACKUP_FILE"
```

### 阶段成果

- 能编写带错误处理的部署脚本。
- 能用 Shell 脚本自动化重复操作。
- 理解 `set -e`、`set -o pipefail` 的作用。

## 阶段 6：性能排查与日志

目标：能快速定位服务器 CPU、内存、磁盘、网络的瓶颈。

### CPU 排查

```bash
top                         # 实时监控，按 P 按 CPU 排序，按 M 按内存排序
vmstat 1 10                 # 每秒输出一次，共 10 次
mpstat -P ALL 1             # 每个 CPU 核心的使用率

# 找到 CPU 最高的进程
ps aux --sort=-%cpu | head -10
```

### 内存排查

```bash
free -h                     # 内存使用概览
cat /proc/meminfo           # 详细内存信息

# 找到内存占用最高的进程
ps aux --sort=-%mem | head -10

# 查看某进程的内存详情
cat /proc/$(pgrep python)/status | grep -i vm
```

### 磁盘排查

```bash
df -h                       # 各分区使用率
du -sh /var/log/*           # 各目录大小
du -sh /* 2>/dev/null | sort -rh | head -10  # 找出最大的目录

iostat -x 1                 # 磁盘 I/O 统计（需安装 sysstat）
iotop                       # 实时 I/O 监控（需安装）
```

### 网络排查

```bash
nethogs                     # 按进程统计网络流量（需安装）
iftop                       # 实时网络带宽（需安装）

# 抓包分析（排查接口问题）
tcpdump -i eth0 port 8000 -w /tmp/capture.pcap
```

### 日志排查

```bash
# 实时追踪应用日志
tail -f /var/log/myapp/app.log

# 过滤最近 1 小时的 ERROR
awk -v d="$(date -d '1 hour ago' '+%Y-%m-%d %H')" '$0 >= d' app.log | grep ERROR

# 统计最多的错误类型
grep "ERROR" app.log | awk '{print $NF}' | sort | uniq -c | sort -rn | head

# systemd 日志
journalctl -u myapp --since "1 hour ago"
journalctl -u myapp -p err   # 只看 error 级别
```

### logrotate 日志切割

```bash
# /etc/logrotate.d/myapp
/var/log/myapp/*.log {
    daily           # 每天切割
    rotate 30       # 保留 30 天
    compress        # 压缩旧日志
    delaycompress   # 延迟一天再压缩
    missingok       # 日志不存在不报错
    notifempty      # 日志为空不切割
    postrotate
        systemctl reload myapp
    endscript
}
```

### 阶段成果

- 能在 5 分钟内定位 CPU/内存/磁盘 的异常来源。
- 能用 `tail -f` + `grep` 快速定位日志中的错误。
- 能配置 logrotate 防止日志撑满磁盘。

## 阶段 7：服务器运维实战

目标：完整走一遍从购买服务器到部署上线的全流程。

### 步骤一：服务器初始化

```bash
# 1. 更新系统
sudo apt update && sudo apt upgrade -y

# 2. 安装基础工具
sudo apt install -y curl wget git vim htop unzip build-essential

# 3. 修改 SSH 端口（安全加固）
sudo vim /etc/ssh/sshd_config
# Port 2222（改为非 22 端口）
# PermitRootLogin no
# PasswordAuthentication no
sudo systemctl restart sshd

# 4. 配置防火墙
sudo ufw allow 2222    # 允许新 SSH 端口
sudo ufw allow 80
sudo ufw allow 443
sudo ufw enable
```

### 步骤二：部署 Python 应用

```bash
# 安装 Python 环境
sudo apt install -y python3.11 python3.11-venv

# 创建应用目录
sudo mkdir -p /opt/myapp
sudo chown deploy:deploy /opt/myapp

# 部署代码
cd /opt/myapp
git clone https://github.com/yourname/myapp.git .
python3.11 -m venv venv
source venv/bin/activate
pip install -r requirements.txt gunicorn uvicorn

# 注册 systemd 服务（见阶段 3）
sudo systemctl enable myapp
sudo systemctl start myapp
```

### 步骤三：配置 HTTPS

```bash
# 安装 Certbot
sudo apt install -y certbot python3-certbot-nginx

# 申请证书（需要域名已解析到服务器 IP）
sudo certbot --nginx -d yourdomain.com

# 证书自动续期（Certbot 会自动配置）
sudo certbot renew --dry-run
```

### 步骤四：监控配置

```bash
# 安装 Prometheus Node Exporter（采集系统指标）
wget https://github.com/prometheus/node_exporter/releases/download/v1.8.0/node_exporter-1.8.0.linux-amd64.tar.gz
tar -xzf node_exporter-*.tar.gz
sudo mv node_exporter-*/node_exporter /usr/local/bin/

# 注册为系统服务并启动
# 然后在 Grafana 导入 Node Exporter 仪表板（ID: 1860）
```

### 阶段成果

- 能独立完成云服务器的初始化、安全加固和应用部署。
- 能配置 HTTPS，让应用用上免费 SSL 证书。
- 能搭建基础监控，异常时收到告警。
